## Grafische Eigenschaften auslesen, &#228;ndern und zur&#252;cksetzen
	
### Grafische Eigenschaften auslesen und &#228;ndern
Auch Eigenschaften wie  <Italic> Farbe </Italic> ,  <Italic> Layer </Italic> ,  <Italic> Sichtbarkeit </Italic>  oder  <Italic> Beschriftungstext </Italic>  eines Objektes k&#246;nnen ausgewertet und ge&#228;ndert werden.
Der Zugriff erfolgt wie auf jede beliebige andere Objekteigenschaft, allerdings in Kategorien oder  <Italic> Sektionen </Italic>  gruppiert, nach dem Muster

```
objekt.SektionN.Schl&#252;ssel=wert
```

Wobei der Name einer Sektion dem Namen in der Konfiguration entspricht und immer mit einer Nummer  <Italic> N </Italic>  endet (mindestens  <Italic> 1 </Italic> ).

<Info>
punkt.Main1.Layer    ...Schl&#252;ssel  <Italic> Ebene </Italic>  der Haupt-Sektion eines Punkt-Objektes.
punkt.Blockl1.Color   ...Schl&#252;ssel  <Italic> Farbe </Italic>  des ersten f&#252;r den Punkt konfigurierten Symbols.
punkt.Block2.Name   ...Schl&#252;ssel  <Italic> Blockname </Italic>  des zweiten f&#252;r den Punkt konfigurierten Symbols.
line.Annotation1.Text ...Schl&#252;ssel  <Italic> Text </Italic>  der ersten Beschriftung des Linienzuges
</Info>


Als  <Italic> wert </Italic>  kann in den meisten F&#228;llen eine Zeichenkette verwendet werden - auch numerische Werte lassen sich als Text darstellen.
So sind z. B. folgende Schreibweisen gleichwertig, um den Farbindex eines Punktsymbols zu &#228;ndern:

```
punkt.Block1.Color=1
bzw.
punkt.Block1.Color=&quot;1&quot;
```


Im Falle einer RGB-Farbe kann der RGB-Wert einfach als Zeichenkette gesetzt werden.

```
punkt.Block1.Color=&quot;24,55,224&quot;
```


<Success>
Welche Sektionen, Schl&#252;ssel und Wertebereiche es gibt, entnehmen Sie bitte der Hilfe zur Konfiguration der entsprechenden Objektklasse.
Zur schnellen Auflistung kann man sich auch des  <Bold> print dir(...) </Bold>  Befehls bedienen:
</Success>


```
print dir(Application.IO.GetObject(&#39;prompt&#39;).Value)
bzw.
print dir(Application.IO.GetObject(&#39;prompt&#39;).Value.Annotation1)
```


<Warning>
&#196;nderungen dieser Eigenschaften gelten immer nur f&#252;r das entsprechende Objekt. Andere Objekte des selben Objekttyps bleiben unver&#228;ndert.
Sollen alle Objekte eines bestimmten Typs anders dargestellt werden, so ist es ratsamer, den Typ selbst in der  <Italic> Konfiguration </Italic>  zu &#228;ndern.
</Warning>


<Warning>
Die Menge an Grafischen Eigenschaften ist haupts&#228;chlich vom  <Italic> Objekttyp </Italic>  abh&#228;ngig.
Folglich kann bei Objekten ohne Typ bzw. mit unbekanntem Typ nicht auf diese Eigenschaften zugegriffen werden.

Aus diesem Grund kann hier auch keine Auto-Vervollst&#228;ndigung w&#228;hrend der Eingabe erfolgen.
</Warning>


### Grafische Eigenschaften auf den Wert der Konfiguration setzen
Setzt man die Eigenschaft auf &quot;None&quot; gilt wieder der Wert aus der Konfiguration.
<Info>
Flaechen = list(Application.Project.GetObjects().WhereObjectClass(&#39;Area&#39;).WhereObjectType(&#39;Grundst&#252;ck Grenzkataster (Nummer mit HWL)&#39;))
for fl in Flaechen:
    fl.AnnotationComplex1.OffsetAbscissa = None
    fl.AnnotationComplex1.OffsetOrdinate = None
    fl.AnnotationComplex1.ReferenceLineActive = 0
</Info>
