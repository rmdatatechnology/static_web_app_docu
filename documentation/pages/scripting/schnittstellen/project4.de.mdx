## Erzeugen von Objekten ohne Eingaben vom Anwender

Es gibt grunds&#228;tzlich die M&#246;glichkeit, Objekte mittels der Methoden unter [Application.Commands](../schnittstellen/commands)
 zu erzeugen.
Dabei wird quasi der Standard-Erzeugebefehl ferngesteuert - f&#252;r s&#228;mtliche Eingaben, die im Skript nicht notiert sind, kann das Programm eine Eingabeaufforderung anzeigen, w&#228;hrend welcher das Skript angehalten wird.
Um Objekte einfach automatisiert zu erzeugen, gibt es daher die Schnittstelle  <Italic>  <Bold> Application.Project </Bold>  </Italic> .

<Success>
Im Gegensatz zu  <Italic> Application.Commands </Italic>  ist es dabei m&#246;glich, Attributwerte gleich bei der Erstellung in Form eines Schl&#252;ssel-Wert  <Italic> Python Dictionary </Italic> &#39;s anzugeben.
</Success>

<Success>
Beim Einf&#252;gen eines Punktes wird die Punktnummer nicht automatisch gesetzt. Das erfolgt nur im Befehl Application.Command.PointNew
</Success>

<Warning>
Automatisiert eingef&#252;gt werden k&#246;nnen lediglich Objekte der Klassen  <Italic> Punkt </Italic> ,  <Italic> Linienzug </Italic> ,  <Italic> Linienzugsymbol </Italic> ,  <Italic> &#39;Fl&#228;che </Italic> ,  <Italic> 2-Punkt Symbol </Italic> , <Italic> 3-Punkt Symbol </Italic>  und  <Italic> Basislinienbema&#223;ung </Italic> :

-  Application.Project.PointNew


-  Application.Project.LinestringNew


- Application.Project.LinestringSymbolNew


-  Application.Project.AreaNew


-  Application.Project.TwoPointSymbolNew


-  Application.Project.ThreePointSymbolNew


-  Application.Project.BaselineDimensionNew




</Warning>

 <Bold> Punkt einf&#252;gen </Bold>  mit den Attributen &quot;Wert&quot; (Text) und &quot;LFD_NR&quot; (Ganzzahl)

```
 <Border>Application.Project.PointNew('Messpunkt',(100,100),{"Wert": "A14", "LFD_NR" : 17})</Border> 
```


 <Bold> Punkt einf&#252;gen </Bold>  mit Namen

```
 <Border>pg = Application.Project.PointNew("Messpunkt",(100,100), {"Internal~Name":"123"}) </Border> 

```


 <Bold> Linienzugsymbol einf&#252;gen </Bold>  entlang aller Linienz&#252;ge vom Typ  <Italic> Fahrbahnrand </Italic> 

```

count=0
for linienzug in list(Application.Project.GetObjects().WhereObjectClass(&#39;LineString&#39;).WhereObjectType(&#39;Fahrbahnrand&#39;)):
    geom = linienzug.Geometry
    for i in range(0, len(geom)-1) :
        (startX,startY,startZ,startB)=geom[i]
        (endX,endY,endZ,endB)=geom[i+1]
        meanX = (startX+endX)/2.0
        meanY = (startY+endY)/2.0
        if Application.Project.LinestringSymbolNew(&#39;Asphaltrand&#39;,(meanX,meanY),linienzug).Ok:
            count+=1
if count &gt; 0:
    print str(count) + &#39; Symbole eingef&#252;gt.&#39; 
```

Die Aufrufe f&#252;r  <Italic> Linestring </Italic>  und  <Italic> Area </Italic>  funktionieren in gleicher Weise.

Die Aufrufe f&#252;r 2- und 3 Punkt Symbole bzw. Basislinienbema&#223;ungen funktionieren in gleicher Weise, jedoch wird folgende Geometrie erwartet:

-  2 Punkt Symbol: Erwartet eine Liste von genau 2 Punkten

-  3 Punkt Symbol: Erwartet eine Liste von genau 3 Punkten

-  Basislinienbema&#223;ung: Erwartet eine Liste von mind. 2 Punkten (1. und 2. Punkt definieren die Basislinie, alle weiteren Punkte sind Zwischenpunkte)
