## Topologie von Objekten auslesen
### Topologie eines Objekts auslesen
&#220;ber die Objekteigenschaft  <Bold> .Topology </Bold>   k&#246;nnen Sie pr&#252;fen, ob verschiedene Objekte nicht nur geometrisch gleich sind sondern sich die selbe Knoten- oder Kantentopologie teilen.
<Success>
Je nach Objektklasse kann es mehrere Ebenen von Topologie geben:

-  Punkthafte Objekte haben eine Knotentopolgie - <Bold> .VertexTopology </Bold> 
-  Linienhafte Objekte haben eine Kantentopologie - <Bold> .SegmentTopology </Bold> 
-  Da Kanten wiederum auf Knoten verweisen, haben solche Objekte auch eine Knotentopologie, die sich aus den St&#252;tzpunkten der Kanten ableitet.




Je nach Kontext kann es notwendig sein, die eine oder die andere Topologie abzufragen.
</Success>



```
...
# Ausgabe in der Form &quot;V&lt;St&#252;tzpunktnummer&gt;&quot;
# punkt1 und punkt1a liegen auf dem selben St&#252;tzpunkt
print punkt1.Topology.VertexTopology    # ergibt V1
print punkt1a.Topology.VertexTopology   # ergibt V1
print punkt2.Topology.VertexTopology    # ergibt V2
```



```
...
linie1=Application.Project.LinestringNew(&#39;Hilfslinie&#39;, [(10, 10), (11, 11), (12, 10, None,-1), (10, 10)]).Object&quot;);
# Ausgabe in der Form &quot;S&lt;Segmentnummer&gt;&quot;
print &#39;SEGMENTS:&#39;,linie1.Topology.SegmentTopology
# Ergibt SEGMENTS: S3,S5,S6
# Ausgabe als Linienverbindung zwischen St&#252;tzpunkten;
# beachte Bogen mit Bulge-Wert
print &#39;VERTICES:&#39;,linie1.Topology.VertexTopology 
# Ergibt VERTICES: LIN[V1-V2],LIN[V2-V4],ARC[V4-B(-1)-V1]&quot;

```


 <Bold> Interpretation </Bold> 
Objekte, die sich die selbe Topologie teilen, haben zwangsl&#228;ufig die selbe Geometrie, nicht aber umgekehrt.

<Warning>
Welche Topologie(n) ein Objekt anbietet, ist bei der Erstellung des Skripts nicht bekannt. Deshalb kann f&#252;r diese Eigenschaft keine Auto-Vervollst&#228;ndigung angeboten werden.
</Warning>

<Warning>
Diese Eigenschaft dient rein zur Datenanalyse und ist schreibgesch&#252;tzt. 
</Warning>
