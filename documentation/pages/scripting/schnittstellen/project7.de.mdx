## Mit  <Bold> Application.Project.Configuration </Bold>  stehen Methoden zur aktuell geladenen Konfiguration bereit.

### Vorhandensein eines Objekttyps pr&#252;fen
Manche Anwendungsf&#228;lle machen es notwendig, dass Objekte eines bestimmten Typs erzeugt werden.
Mittels der Funktion  <Bold>  <Italic> Application.Project.Configuration.HasObjectType(objectClass, objectType) </Italic>  </Bold>  k&#246;nnen Sie pr&#252;fen, ob die
geladenen Konfigurationen des Datenbestands einen bestimmen Objekttyp beinhalten und die Erstellung somit m&#246;glich ist.

Das folgende Beispiel pr&#252;ft, ob der Linienzugtyp  <Italic> Grundst&#252;cksgrenze </Italic>  vorhanden ist.

```
if not Application.Project.Configuration.HasObjectType(&#39;LineString&#39;, &#39;Grundst&#252;cksgrenze&#39;):
    print &#39;Keine Bearbeitung mittels Skript m&#246;glich.&#39; 
```

Zudem kann mit einer Variante des Befehls auch gepr&#252;ft werden, ob der gew&#252;nschte Objekttyp  <Italic> versioniert </Italic>  ist:

```
if not Application.Project.Configuration.HasObjectType(&#39;Point&#39;, &#39;Grenzpunkt&#39;, True):
    print &#39;Keine Bearbeitung mittels Skript m&#246;glich.&#39; 
```


<Warning>
Die Angabe der Objektklasse ist deshalb erforderlich, weil der Typname allein oft nicht eindeutigt ist.
Es gibt z.B. einen  <Italic> Punkt </Italic>  vom Typ  <Italic> Laubbaum </Italic> , ebenso wie ein  <Italic> 2-Punkt-Symbol </Italic>  desselben Namens.
</Warning>

### Verf&#252;gbare Objekttypen auflisten
&#220;ber die Funktion  <Bold>  <Italic> Application.Project.Configuration.GetObjectTypes(objectClass) </Italic>  </Bold>  k&#246;nnen Sie die Namen aller verf&#252;gbaren Objekttypen einer Objektklasse aufz&#228;hlen.

Folgendes Skript kann benutzt werden, um alle Linienzug-Objekte getrennt nach Typ zu bearbeiten:

```
for lsType in Application.Project.Configuration.GetObjectTypes(&#39;LineString&#39;):
    if lsType  &#39;Gasleitung&#39;:
        for obj in Application.Project.GetObjects().WhereObjectType(lsType):
            # Do something...
    elif lsType  &#39;Oberleitung&#39;:
        # Do something else...
```


Des weiteren kann so auch mittels  <Italic> in </Italic>  bzw.  <Italic> not in </Italic>  gepr&#252;ft werden, ob ein bestimmter Typname vorhanden ist:

```
if &#39;Gasleitung&#39; not in Application.Project.Configuration.GetObjectTypes(&#39;LineString&#39;):
    print &#39;Falsche Konfiguration geladen.&#39; 
```


<Success>
Auf dem Ergebnis dieser Funktion k&#246;nnen keine direkten Listen-Operationen durchgef&#252;hrt werden.
Es ist aber wiederum sehr einfach, die Aufz&#228;hlung mittels des  <Italic> Python </Italic> -Befehls  <Italic> list() </Italic>  in eine Liste zu verwandeln.
</Success>


Auch die konfigurierten Objektklassen k&#246;nnen ausgelesen und angezeigt werden:

```
for className in Application.Project.Configuration.GetObjectClasses():
    print className 
```


### Sichtbare und Selektierbare Objekttypen auflisten

```

#Auflisten aller im Darstellungsmanager sichtbar geschaltenen Objekttypen (aus der editierten Datenquelle)
Application.Project.Configuration.VisibleObjectTypes

#Auflisten aller im Darstellungsmanager unsichtbar geschaltenen Objekttypen (aus der editierten Datenquelle)
Application.Project.Configuration.InvisibleObjectTypes

#Auflisten der selektierbaren Objekttypen (aus der editierten Datenquelle)
Application.Project.Configuration.SelectableObjectTypes

#Auflisten der nicht selektierbaren Objekttypen (aus der editierten Datenquelle)
Application.Project.Configuration.NotSelectableObjectTypes

```


### Definierten Objekttyp lesen
Auslesen eines definierten Objekttyps. Liefert &quot;None&quot;, wenn es den Objekttyp nicht gibt.

```

ot = Application.Project.Configuration.GetObjectType(&#39;Linestring&#39;,&#39;Grundst&#252;cksgrenze&#39;)

```


###  Verf&#252;gbare Darstellungen und Ansichten
Mit den folgenden Funktionen ist es m&#246;glich eine Auflistung der in der Datenbank vorhandenen Darstellungen und Ansicht zu bekommen.


```

for item in Application.Project.Configuration.GetViews():
    print item
for item in Application.Project.Configuration.GetRepresentations():
    print item

```


###  Eigenschaften eines Objekttypes
Ein Objekttyp hat mehrere Eigenschaften die ausgelesen werden werden k&#246;nnen:

```

ot = Application.Project.Configuration.GetObjectType(&#39;Linestring&#39;,&#39;Grundst&#252;cksgrenze&#39;)
# Pr&#252;fen ob der Objekttyp derzeit selektierbar ist
print ot.IsSelectable
# Pr&#252;fen ob der Objekttyp versioniert ist
print ot.IsVersioned

```
