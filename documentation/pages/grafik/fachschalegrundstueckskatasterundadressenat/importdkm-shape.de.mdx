## Katastralmappe aus DKM-Shape Daten importieren

<Success>
Der Befehl ist nur in der Fachschale &quot;Grundst&#252;ckskataster und Adressen (AT)&quot; aktiv.
</Success>

DKM-Shape-Daten werden &#252;ber den Befehl  <Menu>Daten importieren</Menu>  in der Werkzeugleiste von  <Placeholder>fullproduct</Placeholder> oder &#252;ber das  <Menu> Menu: [Daten - Daten importieren] </Menu>  eingef&#252;gt.

Die Shape-Dateien werden in eine rmDATA Geodatabase importiert, diese wird danach in das aktuelle Geodokument eingebunden.

Nach Befehlsaufruf wird im Register  <Border>Typauswahl</Border>  die Option  <Border>Katastralmappe aus DKM-SHP (VermV2010) Daten importieren</Border>  bzw. bei alten DKM-Daten die Option  <Border>Katastralmappe aus DKM-SHP (VermV1994) Daten importieren</Border>  gew&#228;hlt.<Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapetypauswahl.png$ImportDKMShapetypauswahl</Image>

Danach werden im Register  <Border>Dateiauswahl</Border>  die zu importierenden Shape-Dateien gew&#228;hlt<Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapedateiauswahl.png$ImportDKMShapedateiauswahl</Image>

Im Register  <Border>Geodatabase-Auswahl</Border>  wird festgelegt, in welche Geodatabase die Dateien importiert werden sollen.<Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapeGeoDBAuswahl.png$ImportDKMShapeGeoDBAuswahl</Image>

In den  <Border>Einstellungen</Border>  wird festgelegt

- ob Nutzungsfl&#228;chen berechnet werden sollen

- ob bestehende DKM-Daten aus der Datenbank gel&#246;scht werden sollen. Diese Option wirkt sich nur aus, wenn in einen bestehende Datenbank importiert wird.




<Warning>
Werden Daten in eine bestehende Datenbank importiert, und wurde das Datenformat ge&#228;ndert (alte Daten sind im Format VermV 1994, neue Daten sind im Format VermV 2010), m&#252;ssen alte Daten aus der Datenbank immer gel&#246;scht werden!
</Warning><Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapeeinstellungen.png$ImportDKMShapeeinstellungen</Image>

Nach  <Border>Fertigstellen</Border>  werden die Daten importiert, ein Transferstatus wird angezeigt.<Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapetransferstatus1.png$ImportDKMShapetransferstatus1</Image><Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapetransferstatus2.png$ImportDKMShapetransferstatus2</Image>

In den  <Border>Einstellungen</Border>  werden nocheinnmal die importierten Objektklassen angezeigt:<Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapeeinstellungen2.png$ImportDKMShapeeinstellungen2</Image>

In den  <Border>erweiterten Einstellungen</Border>  kann die Darstellung und der Defaultma&#223;stab festgelegt werden:<Image>\img\grafik\fachschalegrundstueckskatasterundadressenat\ImportDKMShapeeinstellungenerweitert.png$ImportDKMShapeeinstellungenerweitert</Image>

Der Importvorgang ist nun beendet und die Geodatabase wird ins Projekt eingebunden.
