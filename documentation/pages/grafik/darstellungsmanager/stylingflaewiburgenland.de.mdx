## Vorgangsweise f&#252;r das Einbinden von FL&#196;WI-Shape-Daten des Landes Burgenland

G&#252;ltig f&#252;r Fl&#228;chenwidmungspl&#228;ne erstellt nach der Planzeichenverordnung f&#252;r Digitale Fl&#228;chenwidmungspl&#228;ne 2008 (authentische Kundmachung gem&#228;&#223; LGBl. Nr. 17/2011)

<Success>
F&#252;r die korrekte Darstellung des Fl&#228;chenwidmungsplanes ist im Geodokument von  <Placeholder>fullproduct</Placeholder> das Zielkoordinatensystem &quot;Bundesmeldenetz M34 (EPSG: 31259)&quot; [einzustellen](../einstellungen/dokumenteneinstellungen)
.
</Success>


## Shape-Daten in rmDATA GeoDesktop einbinden

- Im Windows Explorer den Ordner anklicken, in welchem die Shape-Dateien liegen

- Diesen Ordner aus dem Explorer in das Grafikfenster von <Placeholder>fullproduct</Placeholder> ziehen

## Styling zuweisen
Im Darstellungsmangager folgende Schritte durchf&#252;hren:

- Alle Objektklassen des Fl&#228;chenwidmungsplanes markieren

- Kontextmen&#252; aufrufen (Rechtsklick auf eine Objektklasse)

- Im Kontextmen&#252; den Befehl &quot;Styling-Definition aus Datei laden&quot; aufrufen

- Datei &quot;C:\ProgramData\rmDATA\GeoDesktop\Configurations\FL&#196;WI_Burgenland.styling&quot; &#246;ffnen

- Den folgenden Dialog mit OK beenden

# Alternative: Import der Shape-Dateien in eine rmDATA GeoDatabase
Alternativ zum Einbinden und Stylen der Shape-Dateien kann der Fl&#228;chenwidmungsplan auch in eine rmDATA GeoDatabase importiert werden. 
Der Vorteil liegt darin, dass beim Import die neuen Sachdatenfelder &quot;Kategorie&quot;, &quot;Widmung&quot; und &quot;Beschreibung&quot; bef&#252;llt werden und der Anwender damit zu jedem Objekt der Fl&#228;chenwidmung beschreibende Informationen in den Sachdaten vorfindet.
Weiters kann durch &#196;ndern des Ma&#223;stabs die Gr&#246;&#223;e der angeziegten Texte gesteuert werden (Datenquellenmanager - Datenquelle bearbeiten - erweiterte Einstellungen - Ma&#223;stab)

Gehen Sie wie folgt vor:

- Men&#252; Daten - Daten importieren

- Daten aus ESRI-Shape-Dateien importieren

- Die Shape-Dateien f&#252;r den Import ausw&#228;hlen (Anmerkung: Die Dateien m&#252;ssen wie oben angef&#252;hrt bezeichnet sein, auch auf Gro&#223;-Kleinschreibung achten!)

- Als Koordinatensystem der Quelldateien &quot;Bundesmeldenetz M34&quot; w&#228;hlen

- In neue GeoDatabase importieren

- Beim Anlegen der neuen GeoDatabase die Konfiguration &quot;FL&#196;WI_Burgenland.config&quot; verwenden

- Als Koordinatensystem der neuen GeoDatabase sollte das Zielkoordinatensystem des GeoDocuments verwendet werden

- Die neue GeoDatabase speichern

- Die Konvertierungstabelle f&#252;r den Import angeben: 

- &quot;FL&#196;WI_Burgenland_BMN34-Import-Shape.xml&quot;, wenn das Zielkoordinatensystem &quot;Bundesmeldenetz M34&quot; ist

- &quot;FL&#196;WI_Burgenland_GKM34-Import-Shape.xml&quot;, wenn das Zielkoordinatensystem &quot;Landeskoordinaten M34&quot; ist



- Auf <Border>Fertigstellen</Border>  klicken

- Nach dem Import im Dialog &quot;Verbindungseinstellungen&quot; auf das Register &quot;Einstellungen&quot; klicken und <Border>Verwendete Featureklassen auswählen</Border> 
- Im Register &quot;Erweiterte Einstellungen&quot; den Ma&#223;stab auf 1000 &#228;ndern

- <Border>Fertigstellen</Border>
