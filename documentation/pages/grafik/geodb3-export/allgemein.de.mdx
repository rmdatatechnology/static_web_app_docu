## Daten einer rmDATA Geodatabase als private Geodatabase exportieren

Es k&#246;nnen Daten aus einer private GeoDB oder aus einer Enterprise GeoDB (Oracle, SQL-Server) in eine private GeoDB exportiert werden.
Der Befehl kann im Viewingmodus im  <Menu> Menu: [Daten - Daten exportieren] </Menu>  aufgerufen werden.

Nach Befehlsaufruf wird im Register  <Border>Typauswahl</Border>  die Option  <Border>Daten als rmDATA Geodatenbank Datei exportieren</Border>  gew&#228;hlt.<Image>\img\grafik\geodb3-export\geodb3export-typauswahl.png$geodb3export-typauswahl</Image>

Im Register  <Border>Datenquelle</Border>  wird angegeben, welche der verf&#252;gbaren Geodatenbanken exportiert werden soll.<Image>\img\grafik\geodb3-export\geodb3-objekttypen.png$geodb3-objekttypen</Image>

Danach wird im Register  <Border>Dateiauswahl</Border>  die zu exportierenden GeoDB3-Datei angegeben.
Es k&#246;nnen entweder alle Daten aus der Datenbank exportiert werden, oder es werden die Daten f&#252;r den Export in der Grafik selektiert.<Image>\img\grafik\geodb3-export\geodb3-dateiauswahl.png$geodb3-dateiauswahl</Image>

Nach  <Border>Fertigstellen</Border>  wird die Ausdehnung der Datenquelle neu berechnet und der Export durchgef&#252;hrt. Der Transferstatus und eine Zusammenfassung des Exports werden angezeigt:<Image>\img\grafik\geodb3-export\geodb3-transferstatus.png$geodb3-transferstatus</Image>
<Success>
Daten aus einer rmDATA Asset Database k&#246;nnen &#252;ber diese Funktion nicht exportiert werden!
</Success>
