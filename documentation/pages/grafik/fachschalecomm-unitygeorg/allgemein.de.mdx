## Schnittstelle zum Produkt &quot;GeOrg&quot; der Firma Comm-Unity

&#220;ber die Fachschale werden Daten mit dem Produkt &quot;GeOrg&quot; der Firma Comm-Unity ausgetauscht sowie diverse Prozesse und Workflows abgewickelt. Es k&#246;nnen Grundst&#252;cke, Adressen oder beliebige andere Objekte im GIS gesucht werden. Weiters k&#246;nnen auf vorgegebenen Koordinaten Punktobjekte erzeugt und auf diese im GIS gezoomt werden. Zu einem Grundst&#252;ck k&#246;nnen die Anrainer im GIS ermittelt und zur&#252;ck nach GeOrg &#252;bergeben werden.
Werden aktuelle Eigent&#252;mer- oder Adressinformationen von GeOrg bereitgestellt, k&#246;nnen diese im GIS abgerufen werden.

## Fachschale aktivieren
Das Aktivieren der Fachschale erfolgt in den [Dokumenteigenschaften](../einstellungen/dokumenteneinstellungen)


## Einstellungen zur Fachschale
Damit der Datenaustausch zwischen GeOrg und  <Placeholder>product</Placeholder> funktioniert, m&#252;ssen in beiden Produkten gemeinsame Ordner f&#252;r den Dateiaustausch definiert werden. In diesen werden die Dateien f&#252;r den Datenaustausch gelesen und geschrieben.
Die gemeinsamen Ordner werden &#252;ber das  <Menu> Menu: [Dokument - Programmeinstellungen - GeOrg] </Menu>  angegeben:<Image>\img\grafik\fachschalecomm-unitygeorg\georgEinstellungen.png$georgEinstellungen</Image>

### Pfad f&#252;r den Austausch von Objekten
In dem angegebenen Ordner gibt es drei Unterordner:

- Export: Ordner f&#252;r Daten, welche von GeOrg geschrieben und von <Placeholder>product</Placeholder> gelesen werden

- Import: Ordner f&#252;r Daten, welche von <Placeholder>product</Placeholder> geschrieben und von GeOrg gelesen werden

- Metadaten: Ordner f&#252;r Metadaten (z.B. Liste mit Transaktionscodes,...)



### Pfad f&#252;r den Austausch von Adress- und Eigent&#252;merdaten
In diesem Ordner werden die Eigent&#252;merdaten (CSV-Dateien) aus GeOrg gespeichert. Diverse Fachschalen-Befehle greifen auf diese Daten direkt zu.
Weiters werden in diesem Ordner neue GWR-Adress-Dateien gelegt. Beim Start von GeoDesktop werden diese Daten in das Projekt importiert.

## Anrainer ermitteln
Nachdem der Befehl &quot;Anrainer im GIS ermitteln&quot; in GeOrg aufgerufen, das gew&#252;nschte Grundst&#252;ck sowie der gew&#252;nschte Puffer angegeben wurden, wird die entsprechende XML-Datei in den Ordner &quot;Export&quot; gelegt.
Danach ruft der Anwender in  <Placeholder>fullproduct</Placeholder> der Befehl  <Menu>Auftrag übernehmen</Menu>   <Menu> Menu: [Verbindungen] </Menu>  auf.
Das gesuchte Grundst&#252;ck wird gesucht, die Anrainer-Grundst&#252;cke innerhalb des angegebenen Puffers ermittelt und im Grafikfenster angezeigt.
Wurde kein Puffer von GeOrg &#252;bergeben, wird der Puffer in  <Placeholder>product</Placeholder> abgefragt.
Jetzt hat der Anwender die M&#246;glichkeit, die bestehende Selektion durch Klicken auf weitere Grundst&#252;cke zu &#228;ndern.
Klick auf ein selektiertes Grundst&#252;ck deselektiert dieses, Klick auf ein nicht selektiertes Grundst&#252;ck selektiert dieses.
Mit dem Befehl  <Menu>Auftrag übergeben</Menu>  werden die Grundst&#252;cke wieder zur&#252;ck an GeOrg &#252;bergeben, indem eine entsprechende XML-Datei im Ordner &quot;Import&quot; erzeugt wird.
Die von GeOrg generierte XML-Datei wird gel&#246;scht.

Kann das gesuchte Grundst&#252;ck nicht gefunden werden, wird eine entsprechende Meldung angezeigt.

## Objekte mit GeOrg-Eigent&#252;merdaten beschriften
&#220;ber den Befehl  <Menu>Objekte mit GeOrg-Eigentümerdaten beschriften</Menu>  in der Werkzeugleiste von  <Placeholder>fullproduct</Placeholder> oder im  <Menu> Menu: ["Verbindungen"] </Menu>  k&#246;nnen zu einem oder mehreren Objekten mit Grundst&#252;cksnummer die Eigent&#252;mer aus GeOrg als Text angeschrieben werden.
Dazu muss die Grundst&#252;cksnummer als Sachdateninformation beim Objekt vorhanden sein, weiters m&#252;ssen die GeOrg-Eigent&#252;merdaten als Datei im Pfad f&#252;r den Austausch von GeOrg-Adress und Eigent&#252;merdaten vorhanden sein.

Nach Befehlsaufruf wird &#252;ber einen Dialog das Format der Beschriftung festgelegt:<Image>\img\grafik\fachschalecomm-unitygeorg\eigentümerbeschriftenvorlagegeorg.png$eigentümerbeschriftenvorlagegeorg</Image>


-  Eigent&#252;mer-Vorlage: Diese gibt an, welche Informationen in der Beschriftung angef&#252;hrt werden.

-  Weitere Eigent&#252;mer zusammenfassen: Beschr&#228;nkt die Ausgabe von zu vielen Eigent&#252;mern. Ist die maximale Anzahl an Eigent&#252;mern erreicht, wird danach der Ersatztext ausgegeben.

-  Adressen zusammenfassen: Gleiche Adressen von mehreren Eigent&#252;mern werden zusammengefasst.



Es werden die Eigent&#252;mer-Informationen als Redlining-Texte ausgegeben.

## GeOrg Report erstellen
&#220;ber den Befehl  <Menu>GeOrg Report erstellen</Menu>  im  <Menu> Menu: [Verbindungen] </Menu>  kann ein Verzeichnis von Grundst&#252;cken mit Eigent&#252;mern aus GeOrg und einer &#220;bersichtsgrafik der betroffenen Grundst&#252;cke erzeugt werden. Die Eigent&#252;mer-Informationen werden dabei aus dem GeOrg-Austauschverzeichnis gelesen und sind keine amtlichen Daten!

Nach Befehlsaufruf selektiert der Anwender ein oder mehrere Grundst&#252;cke im Grafikfenster.
Wurden vor dem Befehlsaufruf bereits Grundst&#252;cke selektiert, wird der Report mit den selektierten Grundst&#252;cken  erzeugt.

Der erzeugte Report kann gespeichert, gedruckt oder als E-Mail versendet werden


- Speichern: F&#252;r das Speichern stehen folgende Formate zur Vef&#252;gung:

- PDF

- DOC

- XLS



- Drucken: Wird ein Report gedruckt, kann auch eine Kopf- und Fu&#223;zeile f&#252;r den Ausdruck konfiguriert werden (siehe 
[Benutzerdefinierte Einstellungen](../einstellungen/benutzereinstellungen)
)

- Als E-Mail versenden: Es wird der aktuelle Mail-Client ge&#246;ffnet und eine neue E-Mail ge&#246;ffnet. Der Report wird im Format PDF als Anhang eingef&#252;gt.




## Automatischer Import von GWR-Adressen
Bei aktivierter Fachschale &quot;Schnittstelle Comm-Unity: GeOrg&quot; k&#246;nnen bei Bedarf aktualisierte Adressdaten im Format GWR automatisch importiert werden, wenn diese von GeOrg zur Verf&#252;gung gestellt werden.
Beim &#246;ffnen eines GeoDokuments wird im Pfad f&#252;r den Austausch von Adress- und Eigent&#252;merdaten nach einer aktuellen Adress-Datei gesucht. Ist diese neuer als der letzte Adress-Import, wird der Anwender gefragt, ob er die neuere Datei importieren m&#246;chte.
Wenn JA, wird die Adressdatei automatisch in eine Geodatabase importiert, alte Adressen werden aus der Geodatabase gel&#246;scht.
