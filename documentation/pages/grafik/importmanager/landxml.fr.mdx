Sur d&#39;autres pages de param&#233;trisation, vous pouvez d&#233;finir des param&#232;tres sp&#233;cifiques pour le format LandXML.

<Success>
Les &#233;l&#233;ments &quot;CgPoints&quot; et &quot;Surfaces&quot; sont import&#233;s. Pour le &quot;Surfaces&quot; l&#39;importation concerne les donn&#233;es de base (&quot;SourceData&quot;) et les maillages TIN (&quot;Definition&quot;). Les Surfaces qui ne possedent aucune donn&#233;e de base, ne sont pas import&#233;es. Le seul type de maillage support&#233; est le TIN (maillage triangulaire).
</Success>


### Param&#232;tres
Ici on peut d&#233;finir optionnellement d&#39;autres param&#232;tres d&#39;importation.

- <Bold> Param&#232;tres pour l&#39;importation LandXML: </Bold> 
- <Bold> Importer les points comme: </Bold>  Il d&#233;finit si les points import&#233;s depuis les donn&#233;es de base et les CgPoints sont enregistr&#233;s comme simples objets &quot;Point&quot; ou comme &quot;Nuage de points&quot;.

- <Bold> Type d&#39;objets pour points: </Bold>  Il d&#233;finit, &#224; quels types d&#39;objets &quot;Point&quot;, les donn&#233;es de base et les CgPoints import&#233;s, doivent &#234;tre attribu&#233;es.

- <Bold> Type d&#39;objets pour lignes de rupture: </Bold>  Il d&#233;finit, &#224; quels types d&#39;objets &quot;Ligne de rupture&quot;, les donn&#233;es de base import&#233;es, doivent &#234;tre attribu&#233;s.

- <Bold> Type d&#39;objets pour contours de mod&#232;le: </Bold>  Il d&#233;finit, &#224; quels types d&#39;objets &quot;Contours de mod&#232;le&quot;, les donn&#233;es de base import&#233;es, doivent &#234;tre attribu&#233;s.

- <Bold> Type d&#39;objets pour mod&#232;le de terrain: </Bold>  Il d&#233;finit, &#224; quels types d&#39;objets &quot;Mod&#232;le de terrain&quot; les donn&#233;es import&#233;es, doivent &#234;tre attribu&#233;s.



- <Bold> R&#233;solution de conflits: </Bold>  Les noms des points et des surfaces doivent &#234;tre univoques. Dans le cas de noms identiques, il y a les options suivantes:

- <Italic> Renommer </Italic> : Les points et les surfaces import&#233;s sont renomm&#233;s avec un pr&#233;fixe ou un suffixe. Les objets existants sont gard&#233;s tels quels.

- <Italic> Priorit&#233; au point existant </Italic> : Le point existant reste dans le fichier et l&#39;objet import&#233; est ignor&#233;. Le comportement est le m&#234;me que pour <Italic> Renommer </Italic>  sans l&#39;ajout du pr&#233;fixe resp. suffixe.

- <Italic> Priorit&#233; au nouveau point </Italic> : Le nouveau point import&#233; remplace le point existant.

- <Italic> Resoudre int&#233;ractivement </Italic> : Un dialogue de collision s&#39;ouvre, ici vous d&#233;cidez pour chaque point si garder le nouveau ou l&#39;existant dans <Placeholder>product</Placeholder>  





<Warning>
En cas de surfaces avec le m&#234;me nom, les surfaces import&#233;es sont renomm&#233;es avec un pr&#233;fixe resp. un suffixe. Si pr&#233;fixe et suffixe sont vides, la surface n&#39;est pas import&#233;e et l&#39;ancien surface est gard&#233;e.
</Warning>


- <Bold> Protocole: </Bold>  Il d&#233;finit si le protocole d&#233;taill&#233; des objets import&#233;s, doit &#234;tre g&#233;n&#233;r&#233;, ou juste un r&#233;capitulatif.

- <Bold> Importer Region: </Bold>  Il d&#233;finit si pour les points, le champ <Italic> Region </Italic>  doit &#234;tre import&#233; ou ignor&#233;.




Appuyez sur  <Border>Terminer</Border> , pour terminer l&#39;importation. Un dialogue qui affiche l&#39;&#233;tat de l&#39;importation, s&#39;ouvre.
