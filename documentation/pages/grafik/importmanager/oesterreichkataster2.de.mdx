## Setzen Sie spezifische Einstellungen f&#252;r den Import der &#246;sterreichischen digitalen Katastralmappe (DKM) im DXF-Format als Basis f&#252;r Teilungspl&#228;ne.
<GeoMapper> <Menu> Menu: [Datei/ Importmanager] </Menu> </GeoMapper><GeoDesigner> <Menu> Menu: [Datei/ Import] </Menu> </GeoDesigner>

Mit dem Import wird die Katastralmappe als Basis f&#252;r Teilungspl&#228;ne importiert. Dabei werden alle Grenzen gem&#228;&#223; dem Zeichenschl&#252;ssel zur Vermessungverordnung als &#252;bernommene Grenzen importiert und somit strichliert dargestellt.

### Einstellungen
Hier k&#246;nnen Sie weitere Import Einstellungen setzen, diese Einstellungen sind optional.

- <Bold> Sonstiges: </Bold>  

-  KG in Grenzpunktnummern &#252;bernehmen: Gibt an, ob bei Grenzpunkten die Region &#252;bernommen wird oder nicht.





<Success>
Bei Triangulierungs- und Einschaltpunkten bleibt die Region beim Import immer erhalten. Nur damit ist eine m&#246;gliche Verbindung zu rmGEO4  korrekt.
</Success>


-  Bereinigung der Knoten/Kanten-Struktur: Gibt an, ob nach dem Import der Befehl &quot;Knoten/Kanten Struktur bereinigen&quot; gestartet werden soll

-  Bereinigung der Mappenblattschnitte: Gibt an, ob nach dem Import der Befehl &quot;Mappenblattschnitte entfernen&quot; gestartet werden soll

- Grenzen darstellen als: 

- gegenst&#228;ndlich: Linien der Grenzen werden durchgezogen dargestellt

- &#252;bernommen: Linien der Grenzen werden strichliert gezeichnet (Defaultwert)








- <Bold> Konfliktaufl&#246;sung: </Bold> Punkt- und Fl&#228;chenbezeichnungen m&#252;ssen eindeutig sein. Es stehen folgende Optionen bei Namensgleichheit  zur Verf&#252;gung:

- <Italic> Umbenennen </Italic> : Die importiereten Punkte und Fl&#228;chen werden mit dem Pr&#228;fix bzw. Suffix umbenannt. Die bestehenden Objekte bleiben unver&#228;ndert erhalten.

- <Italic> Vorrang bestehender Punkt </Italic> : Der in der Datei existierende Punkt bleibt bestehen, das Objekt aus der Quelle wird ignoriert. Das Verhalten ist das selbe wie bei <Italic> Umbenennen </Italic>  ohne Angabe eines Pr&#228;fix bzw. Suffix.

- <Italic> Vorrang neuer Punkt </Italic> : Der neu importierte Punkt ersetzt den bestehenden Punkt.

- <Italic> Interaktiv l&#246;sen </Italic> : Sie erhalten einen Kollisionsdialog, in dem Sie f&#252;r jeden Punkt entscheiden, ob die Punkte aus <Placeholder>product</Placeholder> oder aus der Quelle verwendet werden sollen. 


<Warning>
Bei gleichnamigen Fl&#228;chen werden die importierten Fl&#228;chen mit dem Pr&#228;fix bzw. Suffix umbenannt. Sind die beiden Werte leer, wird die Fl&#228;che nicht importiert. Es bleibt die alte Fl&#228;che erhalten.

</Warning>


- <Bold> Protokollierung: </Bold>  Legt fest, ob ein detailliertes Protokoll f&#252;r die importierten Objekte generiert werden soll, oder nur eine Zusammenfassung.




Dr&#252;cken Sie  <Border>Fertigstellen</Border> , um den Import abzuschlie&#223;en. Es wird ein Dialog ge&#246;ffnet, der den Status des Imports anzeigt.
