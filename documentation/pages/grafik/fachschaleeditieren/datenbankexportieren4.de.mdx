## Daten aus einer GeoDB als Autodesk DXF Datei exportieren

Mit dem Export werden Daten der aktuell zum Bearbeiten ge&#246;ffneten Datenbank in eine Autodesk DXF-Datei exportiert.

Rufen Sie den Befehl im  <Menu> Menu: [Datenbank - Datenbank exportieren] </Menu>  auf.

## Typauswahl
Im Register &quot;Typauswahl&quot; w&#228;hlen Sie  <Border>Daten als Autodesk DXF Datei exportieren</Border> 

## Dateiauswahl
Geben Sie die Zieldatei an.
W&#228;hlen sie die Objekte, die Sie exportieren m&#246;chten.
W&#228;hlen Sie das Koordinatensystem der Zieldatei
<Success>
Siehe auch [Allgmeines zum Export aus einer Datenbank](../fachschaleeditieren/datenbankexportieren)
</Success>

## Einstellungen<Image>\img\grafik\fachschaleeditieren\dwgexport-einstellungen.png$dwgexport-einstellungen</Image>


- Dateiversion: Geben Sie das gew&#252;nschte Autodesk-Dateiformat an. Es stehen die Formate 

- AutoCAD 2013

- AutoCAD 2010

- AutoCAD 2007 zur Verf&#252;gung.



- Dezimaltrennzeichen: Geben Sie das Dezimaltrennzeichen an, das z.B. bei Blockattributen verwendet werden soll.

- Vorlagendatei exportieren

- vollst&#228;ndig: Alle Definitionen (Bl&#246;cke, Layer, Linienstile, Textstile,...) der Vorlagenzeichnung werden exportiert.

- nur verwendete Definitionen: Es werden nur jene Definitionen exportiert, welche f&#252;r die Darstellung der exportierten Daten in AutoCAD ben&#246;tigt werden.



- N&#228;herung von B&#246;gen

- Nein: B&#246;gen werden als B&#246;gen exportiert

- Ja: B&#246;gen werden mit Liniensegmenten angen&#228;hert.



- Maximale Pfeilh&#246;he bei Bogeninterpolation: Werden B&#246;gen interpoliert, bestimmt die Maximale Pfeilh&#246;he die Anzahl der erzeugten Segmente.

- Objekte 3D exportieren

- Ja: H&#246;heninformationen von Punkten werden auf Symbole und Linienz&#252;ge &#252;bertragen. Die entsprechenden Objekte werden mit Z-Koordinate bzw. als 3D-Polylinien exportiert. 



- H&#246;henlinien exportieren

- als 3D Polylnien: H&#246;henlinien aus rmDATA GeoModeller werden als 3D-Polylinien exportiert

- als 2D Polylnien mit Erhebungen: Die H&#246;he von H&#246;henlinien aus rmDATA GeoModeller werden als Erhebung exportiert



- CAD-Elemente exportieren: Option f&#252;r rmDATA GeoMapper

- Unsichtbare Beschriftungen exportieren

- Ja: Ausgeblendete Beschriftungen werden z.B. als Blockattribute exportiert

- Nein: Ausgeblendete Beschriftungen werden nicht exportiert, auch nicht als Blockattribute



- Leere Attribute exportieren

- Nein: Leere Attribute werden nicht exportiert.



- Linienz&#252;ge ausgespart exportieren: Beim 2D-Export werden Linienz&#252;ge an den Punktsymbolen ausgespart.




## Export
Mit  <Border>Fertigstellen</Border>  werden die Daten aus der Datenbank in die DWG-Datei exportiert.
<Success>
Als Vorlagezeichnung f&#252;r den Export wird die aktuelle Prototypzeichnung der ersten in der GeoDatenbank verwendeten Konfiguration verwendet. Alle Einstellungen (Einheiten, Systemvariablen,...) und Layouts dieser Vorlagenzeichnung finden sich damit auch in der neuen exportierten Datei. Bestehende Grafik-Elemente des Modellbereichs der Vorlagenzeichnung werden jedoch gel&#246;scht. 
</Success>
