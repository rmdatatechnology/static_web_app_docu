### AutoWiederherstellen
W&#228;hrend des Arbeitens in einer Private Geodatabase werden in einem vorgegebenen Zeitintervall alle &#196;nderungen in eine tempor&#228;re Datei gespeichert. Damit verlieren Sie keine Daten, sollte  <Placeholder>fullproduct</Placeholder> einmal abst&#252;rzen. Beim neuerlichen &#214;ffnen der Geodatabase im Editiermodus werden Sie gefragt, ob Sie die nicht gespeicherten Daten wieder herstellen m&#246;chten.  <Placeholder>product</Placeholder> &#246;ffnet und l&#228;dt danach den zuletzt editierten Bereich.
Wenn Sie auf [Nein] klicken, dann werden die tempor&#228;ren Daten gel&#246;scht.

Zudem kann man das Interval f&#252;r das erstellen der Wiederherstellungdatei angeben. Dieses kann nicht unter 30 Sekunden eingestellt werden.


### Konfigurations&#228;nderungen wiederherstellen
W&#228;hrend des Arbeitens an Projekten k&#246;nnen sie implizit die Konfiguration des Projektes bearbeiten. Diese Einstellung gibt an, ob die &#196;nderungen an der Konfiguration in der Wiederherstellungsdatei mitgesichert werden. 

<Success>
In einer Enterprise-GeoDatabase kann die Option &quot;Auto Wiederherstellen&quot; nicht angewendet werden, alle &#196;nderungen m&#252;ssen vom Anwender manuell gespeichert werden. 
</Success>
