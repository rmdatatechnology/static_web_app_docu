## Gestione della conversione di elementi CAD in polilinee nel Geodatabase.
Le polilinee possono essere create da diversi tipi di elementi CAD lineari.

```
La creazione delle polilinee &#232; definita tramite l&#39;elemento  <Bold> &lt;LineStrings&gt; </Bold> .
Si tratta di una raccolta di elementi  <Bold> &lt;LineString&gt; </Bold>  che gestiscono la conversione:
 <Bold> &lt;LineStrings&gt; </Bold> 
   &lt;LineString Name=&quot;Esempio1&quot;&gt;
       <Italic> &lt;!-- Definizione --&gt; </Italic> 
   &lt;/LineString&gt;
    <Italic> &lt;!-- Ulteriori elementi &lt;LineString&gt; --&gt; </Italic> 
 <Bold> &lt;/LineStrings&gt; </Bold> 
```

### &lt;LineString&gt;
### 
# Attributi


- <Bold> Name </Bold>  (obbligatorio)

-  Definisce il nome del tipo di linea destinazione nel geodatabase. Tutti gli elementi CAD considerati dal filtro sono convertiti in questo tipo di linea.





### 
# Elementi

Con i seguenti elementi si possono definire gli oggetti da cui sono create le polilinee.

1. <Bold> &lt;GraphicAttributeSettings&gt; </Bold>  (da 0 a 1)

- Descrive la ripresa di propriet&#224; grafiche dagli elementi CAD. Vedere 
[Elemento GlobalSettings](../anhang/formatbeschreibungmappingautodesk2)
.



2. <Bold> &lt;Line&gt; </Bold>  (da 0 a 1)

- Gestisce la conversione delle linee.



3. <Bold> &lt;Polyline&gt; </Bold>  (da 0 a 1)

- Gestisce la conversione delle polilinee.



4. <Bold> &lt;Arc&gt; </Bold>  (da 0 a 1)

- Gestisce la conversione degli archi.





<Success>
Se descritto, al posto di un nome completo, possono anche essere usate delle espressioni regolari ( <Italic> RegEx </Italic> ). Per immettere una espressione regolare, usare il prefisso  <Bold> re: </Bold>  
</Success>


### 
#  Creare poligonali dalle linee (&lt;Line&gt;)



```
Struttura di una mappatura Linea-a-Poligonale

 <Bold> &lt;Line&gt; </Bold> 
   <Bold> &lt;Filter&gt; </Bold> 
     <Italic> &lt;!-- Descrive come devono essere filtrate le linee. --&gt;
          Tutti gli elementi seguenti sono facoltativi; 
          gli elementi presenti sono collegati tra loro con AND,
          quindi tutti devono essere soddisfatti --&gt; </Italic> 
     <Bold> &lt;Color&gt; </Bold>  <Italic> &lt;!-- Numero intero, colore indice --&gt; </Italic>  <Bold> &lt;/Color&gt; </Bold> 
     <Bold> &lt;Layer&gt; </Bold>  <Italic> &lt;!-- Nome risp. RegEx --&gt; </Italic>  <Bold> &lt;/Layer&gt; </Bold> 
   <Bold> &lt;/Filter&gt; </Bold> 
 <Bold> &lt;/Line&gt; </Bold> 

```

```
Struttura di una mappatura Arco-a-Poligonale

 <Bold> &lt;Arc&gt; </Bold> 
   <Bold> &lt;Filter&gt; </Bold> 
     <Italic> &lt;!-- Descrive come devono essere filtrate le linee. --&gt;
          Tutti gli elementi seguenti sono facoltativi;
```

<Info>
Esempio:

- Si devono creare delle poligonali in geodatabase del tipo <Bold> Grundst&#252;cksgrenze </Bold> .


- Per le propriet&#224; grafiche devono essere usate quelle definite per il tipo di oggetto nella configurazione (non riprese).


- La geometria viene implicitamente creata dalla geometria delle linee.


- Questa poligonali devono risultare da:


-  Linee sui layer <Bold> N-GG </Bold>  risp. <Bold> GG </Bold> 
-  Polilinee sui layer <Bold> N-GG </Bold>  risp. <Bold> GG </Bold>  


-  Archi sui layer <Bold> N-GG </Bold>  risp. <Bold> GG </Bold>
