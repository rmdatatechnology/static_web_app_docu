## Steuern Sie die &#220;bernahme von CAD-Elementen als Fl&#228;che in der Geodatenbank.
Fl&#228;chen k&#246;nnen sowohl aus punkthaften als auch aus linienhaften oder fl&#228;chigen Elementen erzeugt werden.

```
Die Erzeugung von Fl&#228;chen wird &#252;ber das Element  <Bold> &lt;Areas&gt; </Bold>  definiert.
Es ist eine Sammlung von  <Bold> &lt;Area&gt; </Bold>  Elementen, welche die Konvertierung steuern:
 <Bold> &lt;Areas&gt; </Bold> 
   &lt;Area Name=&quot;Beispiel1&quot;&gt;
       <Italic> &lt;!-- Definition --&gt; </Italic> 
   &lt;/Area&gt;
    <Italic> &lt;!-- Weitere &lt;Area&gt; Elemente --&gt; </Italic> 
 <Bold> &lt;/Areas&gt; </Bold> 
```

### &lt;Area&gt;
### 
# Attribute


- <Bold> Name </Bold>  (obligat)

-  Bezeichnet den Namen des Ziel-Fl&#228;chentyps in der Geodatenbank. S&#228;mtliche durch den/die Filter betroffenen CAD-Element werden in diesen Fl&#228;chentyp konvertiert.





### 
# Elemente

Mit folgenden Elementen kann definiert werden, woraus Fl&#228;chen erzeugt werden.

1. <Bold> &lt;GraphicAttributeSettings&gt; </Bold>  (0 bis 1)

- Beschreibt die &#220;bernahme grafischer Eigenschaften aus CAD-Elementen. Siehe 
[GlobalSettings-Element](../anhang/formatbeschreibungmappingautodesk2)
.



2. <Bold> &lt;Insert&gt; </Bold>  (0 bis 1)

- Steuert die Konvertierung von CAD-Bl&#246;cken. Siehe 
[Punkte aus CAD-Bl&#246;cken erzeugen ](../anhang/formatbeschreibungmappingautodesk4)
.



3. <Bold> &lt;Text&gt; </Bold>  (0 bis 1)

- Steuert die Konvertierung von Texten. Siehe 
[Punkte aus Texten erzeugen ](../anhang/formatbeschreibungmappingautodesk7)
.



4. <Bold> &lt;Point&gt; </Bold>  (0 bis 1)

- Steuert die Konvertierung von CAD-Punkten ( <Italic> POINT </Italic> ). Siehe 
[Punkte aus Punkt-Elementen erzeugen ](../anhang/formatbeschreibungmappingautodesk4)
.



5. <Bold> &lt;Hatch&gt; </Bold>  (0 bis 1)

- Steuert die Konvertierung von Schraffuren.



6. <Bold> &lt;Polyline&gt; </Bold>  (0 bis 1)

- Steuert die Konvertierung von Polylinien.





<Success>
Wo beschrieben, k&#246;nnen anstatt ganzer Namen auch Regul&#228;re Ausdr&#252;cke ( <Italic> RegEx </Italic> ) verwendet werden. Um einen Regul&#228;ren Ausdruck einzuleiten, verwenden Sie das Pr&#228;fix  <Bold> re: </Bold>  
</Success>

### 
#  Fl&#228;chen aus Schraffuren erzeugen (&lt;Hatch&gt;)



```
Struktur eines Schraffur-zu-Fl&#228;che Mappings

 <Bold> &lt;Hatch&gt; </Bold> 
   <Bold> &lt;Filter&gt; </Bold> 
     <Italic> &lt;!-- Beschreibt, wonach Schraffuren gefiltert werden sollen. --&gt;
          Jedes der folgenden Elemente ist optional; 
          vorhandene Elemente werden mit UND verkn&#252;pft,
          d. h. alle m&#252;ssen zutreffen --&gt; </Italic> 
     <Bold> &lt;Color&gt; </Bold>  <Italic> &lt;!-- Ganzzahl, d. i. Farbindex --&gt; </Italic>  <Bold> &lt;/Color&gt; </Bold> 
     <Bold> &lt;Layer&gt; </Bold>  <Italic> &lt;!-- Name bzw. RegEx --&gt; </Italic>  <Bold> &lt;/Layer&gt; </Bold> 
   <Bold> &lt;/Filter&gt; </Bold> 
 <Bold> &lt;/Hatch&gt; </Bold> 

```

<Info>
Beispiel:

- Es sollen Geodatenbank-Fl&#228;chen vom Typ <Bold> Fl&#228;chenschraffur </Bold>  erzeugt werden.


- Als Fl&#228;chenfarbe soll die Schraffurfarbe verwendet werden.


- Die Geometrie wird implizit aus der Umrandung der Schraffur erzeugt.


- Diese Fl&#228;chen sollen aus Schraffuren auf Layern, die mit <Bold> Schraffur </Bold> _ beginnen, entstehen.
