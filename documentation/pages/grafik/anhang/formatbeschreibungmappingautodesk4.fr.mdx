## Gestion da la conversion d&#39;&#233;l&#233;ments CAD vers points dans la G&#233;odatabase.

```
La cr&#233;ation de points est d&#233;finie par l&#39;&#233;l&#233;ment  <Bold> &lt;Points&gt; </Bold> .
Il s&#39;agit d&#39;une collection d&#39;&#233;l&#233;ments  <Bold> &lt;Point&gt; </Bold> , qui g&#233;re la conversion:
 <Bold> &lt;Points&gt; </Bold> 
   &lt;Point Name=&quot;Exemple1&quot;&gt;
       <Italic> &lt;!-- Definition --&gt; </Italic> 
   &lt;/Point&gt;
    <Italic> &lt;!-- Ult&#233;rieurs &#233;l&#233;ments &lt;Point&gt; --&gt; </Italic> 
 <Bold> &lt;/Points&gt; </Bold> 
```

### &lt;Point&gt;
### 
# Attributs


- <Bold> Name </Bold>  (obligatoire)

- Il d&#233;finit le nom du type de point de destination dans la g&#233;odatabase. Tous les &#233;l&#233;ments CAD s&#233;lectionn&#233;s par le(s) filtre(s) sont convertis en ce type de point.





### 
# El&#233;ments

Avec les &#233;l&#233;ments suivants on peut d&#233;finir de quels objets les points sont cr&#233;&#233;s.

1. <Bold> &lt;GraphicAttributeSettings&gt; </Bold>  (de 0 &#224; 1)

- Il d&#233;finit la reprise des propri&#233;t&#233;s graphiques depuis les &#233;l&#233;ments CAD. Voir 
[El&#233;ment GlobalSettings](../anhang/formatbeschreibungmappingautodesk2)
.



2. <Bold> &lt;Insert&gt; </Bold>  (de 0 &#224; 1)

- Il g&#232;re la conversion de blocs CAD.



3. <Bold> &lt;Text&gt; </Bold>  (de 0 &#224; 1)

- Il g&#232;re la conversion de textes.



4. <Bold> &lt;Point&gt; </Bold>  (de 0 &#224; 1)

- Il g&#232;re la conversion de points CAD ( <Italic> POINT </Italic> ).





<Success>
Si d&#233;crit, au lieu de noms complets on peut utiliser aussi les expressions r&#233;guli&#232;res ( <Italic> RegEx </Italic> ). Pour ins&#233;rer une expression r&#233;guli&#232;re, utilisez le pr&#233;fixe  <Bold> re: </Bold>  
</Success>


<Info>

<th>Description</th>

</tr>

# Cr&#233;er points depuis blocs CAD (&lt;Insert&gt;)



```
Structure d&#39;un mappage Bloc-vers-Point

 <Bold> &lt;Insert&gt; </Bold> 
   <Bold> &lt;Filter&gt; </Bold> 
     <Italic> &lt;!-- Il d&#233;finit selon quoi les blocs-source doivent &#234;tre filtr&#233;s. --&gt;
          Chacun des &#233;l&#233;ments qui suit est facultatif; 
          les &#233;l&#233;ments existants sont li&#233;s avec AND,
          c&#39;est &#224; dire que tous doivent &#234;tre valables --&gt; </Italic> 
     <Bold> &lt;Color&gt; </Bold>  <Italic> &lt;!-- Nombre entier, index des couleurs --&gt; </Italic>  <Bold> &lt;/Color&gt; </Bold> 
     <Bold> &lt;Layer&gt; </Bold>  <Italic> &lt;!-- Nom resp. RegEx --&gt; </Italic>  <Bold> &lt;/Layer&gt; </Bold> 
     <Bold> &lt;Blockname&gt; </Bold>  <Italic> &lt;!-- Nom resp. RegEx --&gt; </Italic>  <Bold> &lt;/Blockname&gt; </Bold> 
   <Bold> &lt;/Filter&gt; </Bold>

     <Bold> &lt;Layer&gt; </Bold>  <Italic> &lt;!-- nom resp. RegEx; facultatif --&gt; </Italic>  <Bold> &lt;/Layer&gt; </Bold> 
     <Bold> &lt;ACADAttribute Name </Bold> &quot;AttName <Bold> &quot; Tag </Bold> &quot;AttTag <Bold> &quot; Target  </Bold> &quot;AttTarget <Bold> &quot; &gt; </Bold>   <Italic> &lt;!-- pour chaque attribut mapp&#233; --&gt; </Italic> 
       <Bold> &lt;Tag&gt; </Bold>  <Italic> Attibution espace r&#233;serv&#233;; facultatif </Italic>  <Bold> &lt;/Tag&gt; </Bold> 
       <Bold> &lt;Value&gt; </Bold>  <Italic> Attibution espace r&#233;serv&#233;; facultatif </Italic>  <Bold> &lt;/Value&gt; </Bold> 
       <Bold> &lt;Name&gt; </Bold>  <Italic> Attibution espace r&#233;serv&#233;; facultatif </Italic>  <Bold> &lt;/Name&gt; </Bold> 
       <Bold> &lt;Layer&gt; </Bold>  <Italic> Attibution espace r&#233;serv&#233;; facultatif </Italic>  <Bold> &lt;/Layer&gt; </Bold> 
     <Bold> &lt;/ACADAttribute&gt; </Bold> 
     <Bold> &lt;NG&gt; </Bold> 
       <Italic> &lt;!-- Attributs de destination dana la G&#233;odatabase --&gt; </Italic> 
       <Italic> &lt;!-- Ici on peut ins&#233;rer des valeurs discr&#232;tes ou 
            des espaces r&#233;serv&#233;es depuis un ACADAttribute --&gt; </Italic> 
       <Bold> &lt;Attribut Name=&quot; </Bold> Attributname1 <Bold> &quot;&gt; </Bold>  valeur resp. cha&#238;ne de format <Bold> &lt;/Attribut&gt; </Bold> 
     <Bold> &lt;/NG&gt; </Bold> 
   <Bold> &lt;/AttributeMapping&gt; </Bold> 
 <Bold> &lt;/Insert&gt; </Bold> 

```


<Success>
&lt;ACADAttribute Name <Bold> &quot;AttName </Bold> &quot; Tag <Bold> &quot;AttTag </Bold> &quot; Target  <Bold> &quot;AttTarget </Bold> &quot; &gt; dans cet &#233;l&#233;ment les attributs Name, Tag et Target sont facultatifs. Name et Tag peuvent &#234;tre utilis&#233;s pour identifier un attribut bien d&#233;termin&#233;. Target d&#233;finit la destination dans laquelle les propri&#233;t&#233;s graphiques sont reprises (AttributeSettings). Si on utilise Target=&quot;Annotation3&quot; la Couleur/Position/etc. est reprise dans l&#39;&#233;tiquette 3.
</Success>


<Warning>
Les espaces r&#233;serv&#233;s peuvent pour chacun &#234;tre extraits, entre parenth&#232;ses boucl&#233;es, depuis une cha&#238;ne de caract&#232;res d&#39;un  <Bold> &lt;ACADAttribute&gt; </Bold>  et utilis&#233;s dans un  <Bold> &lt;Attribut&gt; </Bold> . Aussi plusieurs espaces r&#233;serv&#233;s peuvent &#234;tre extraits depuis un attribut CAD.
</Warning>


<Info>
L&#39;exemple suivant doit &#234;tre lu comme suit:

-  Il faut cr&#233;er des points g&#233;odatabase du type <Bold> Triangulierungspunkt (TP) - Kirche </Bold> .


-  Rotation et position doivent <Italic> g&#233;n&#233;ralement </Italic>  &#234;tre reprises depuis d&#39;&#233;l&#233;ments CAD.


-  Ces points doivent r&#233;sulter depuis blocs nomm&#233;s <Bold> FIG012 </Bold>  sur les layers <Bold> N-TP </Bold>  resp. <Bold> TP </Bold> .


-  Les attibuts de bloc doivent &#234;tre repris de la mani&#232;re suivante:


-  Depuis le <Italic> Tag </Italic>  (Marke) de l&#39;attribut de bloc <Bold> Punktnummer </Bold>  est extrait un espace r&#233;serv&#233; nomm&#233; <Bold> KZ </Bold> .


-  Depuis cette valeur, s&#233;par&#233; par le signe moins &#39;-&#39; 2 espaces r&#233;serv&#233;s <Bold> Name </Bold>  et <Bold> Region </Bold> .




-  Les valeurs ainsi obtenues doivent &#234;tre transf&#233;r&#233;s dans des attributs de la G&#233;odatabase.


- <Italic> La valeurs du Name </Italic>  comme Nom de point


- <Italic> La valeurs du Region </Italic>  comme R&#233;gion de point


- <Italic> La valeurs du KZ </Italic>  comme Subname de point.
