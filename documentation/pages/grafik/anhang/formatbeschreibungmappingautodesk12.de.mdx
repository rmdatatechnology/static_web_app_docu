## Nutzen Sie erweiterte Funktionen der Mappingtabelle, um Geodaten effektiv aus CAD-Datenbest&#228;nden zu generieren.
Wie in den Kapiteln zur [Zuweisung](../anhang/formatbeschreibungmappingautodesk3)
 beschrieben, k&#246;nnen Sie pro Geodatenbank-Klasse bestimmen, welche CAD-Elemente f&#252;r die Konvertierung in Frage kommen. Dabei wird grunds&#228;tzlich aus 1 CAD-Element jeweils

-  genau 1 Geodaten-Objekt

-  in der aktiven Version des Projekts



generiert.

Durch Sonderfunktionen der Mappingtabelle kann diese Art der Objektgenerierung ge&#228;ndert werden.

## 1 Objekt aus mehreren Elementen zusammensetzen 
Die Mappingtabelle bietet die M&#246;glichkeit, mehrere r&#228;umlich zusammenfallende CAD-Elemente auf 1 Objekt abzubilden.
Diese Funktion kann grunds&#228;tzlich f&#252;r alle Arten von Zielobjekten verwendet werden, eignet sich aber z. B. besonders daf&#252;r, Symbole aus mehreren CAD-Bl&#246;cken zusammenzuf&#252;gen.

### Syntax 
<Info>
Die Erzeugung 1 Objekts aus mehreren Elementen wird &#252;ber das XML-Element  <Bold> &lt;MultipleObjectMapping&gt; </Bold>  und einer Aufz&#228;hlung von  <Bold> &lt;Objects&gt; </Bold>  definiert. Es kann als Unter-Element s&#228;mtlicher [Objektklassen-Elemente](../anhang/formatbeschreibungmappingautodesk3)
 verwendet werden.
</Info>

<Warning>
Die Konvertierung erfolgt nur dann, wenn alle Objekte anhand des Filters gefunden werden.
</Warning>


<Info>
Beispiel:

-  Der Punkt vom Typ <Bold> Grenzpunkt (GP) nicht gekennzeichnet </Bold>  soll zusammengesetzt werden...


-  Aus 1 Block mit Namen <Bold> FIG024 </Bold>  auf dem Layer <Bold> GP2 </Bold>  und


-  einem Block <Bold> FIG025 </Bold>  auf dem Layer <Bold> GP2 </Bold> 


-  Beide Bl&#246;cke m&#252;ssen um den selben Einsetzpunkt liegen.


-  Die Attribute des Zielobjekts werden allesamt aus den Blockattributen des ersten Blocks (FIG024) &#252;bernommen.

</Info>


## Objekte in mehreren Planversionen erstellen 
Pro CAD-Element k&#246;nnen mehrere verwandte Objekte in unterschiedlichen Versionen des Plans erzeugt werden.
Dies ist insbesondere dann n&#252;tzlich, wenn Sie versionierte CAD-Ausgangsdaten, wie z. B. Teilungspl&#228;ne, &#252;bernehmen m&#246;chten.
Objekte k&#246;nnen z. B. in einer Version als &quot;neu&quot; und in einer Folgeversion als &quot;gel&#246;scht&quot; generiert werden, was zu einer Streichung f&#252;hrt.

### Syntax

```
Die Erzeugung mehrerer Versionen eines Objekts wird &#252;ber das XML-Element  <Bold> &lt;VersioningSettings&gt; </Bold>  definiert.
Die enthaltenen Elemente steuern die Versionen anhand von Layern und gelten global f&#252;r die gesamte Konvertierung:

 <Bold> &lt;VersioningSettings&gt; </Bold> 
   <Bold> &lt;Prefix Name </Bold> =&quot;Layer1&quot; <Bold> &gt; </Bold> 
     <Bold> &lt;Create&gt; </Bold>  <Italic> &lt;!-- Name der Version, in der das Objekt &quot;neu&quot; ist (optional) --&gt; </Italic>  <Bold> &lt;/Create&gt; </Bold> 
     <Bold> &lt;Remove&gt; </Bold>  <Italic> &lt;!-- Name der Version, in der das Objekt &quot;gel&#246;scht&quot; ist  (optional) --&gt; </Italic>  <Bold> &lt;/Remove&gt; </Bold> 
   <Bold> &lt;/Prefix&gt; </Bold> 
   <Italic> &lt;!-- Weitere Prefix-Eintr&#228;ge --&gt; </Italic> 
 <Bold> &lt;/VersioningSettings&gt; </Bold> 

```

<Warning>
Achten Sie darauf, dass gel&#246;schte Objekte keinen Nachfolger in einer sp&#228;teren Version erhalten.
</Warning>

<Info>
Beispiel:

- F&#252;r jegliche CAD-Elemente auf Layern, die mit <Bold> L1 </Bold>  beginnen,...


- wird 1 <Italic> neues </Italic>  Objekt in der Version <Bold> alt </Bold>  erstellt...


- sowie 1 <Italic> gel&#246;schtes </Italic>  Objekt in der Version <Bold> berichtigt </Bold> .


- Dies f&#252;hrt dazu, dass die entsprechenden Objekte in der Version <Italic> berichtigt </Italic>  gestrichen erscheinen und in der Folgeversion nicht sichtbar ist.




- F&#252;r CAD-Elemente auf Layern, die mit <Bold> N3 </Bold>  beginnen, wird 1 Objekt in der Version <Bold> neu </Bold>  erzeugt. In einer fr&#252;heren Version des Plans sind diese Objekte nicht vorhanden und daher nicht sichtbar.

</Info>
