## ALKIS-Daten aus NAS-Dateien importieren

<Success>
Der Befehl ist nur in der Fachschale &quot;Liegenschaftskataster (DE)&quot; aktiv.
</Success>


Der Befehl  <Menu>Daten aus NAS-Dateien importieren</Menu>  kann &#252;ber das  <Menu> Menu: [Daten - Daten importieren] </Menu>  aufgerufen werden.

Es werden ALKIS-NAS-Dateien in eine rmDATA Geodatabase importiert.

ALKIS-NAS-Daten werden &#252;ber den Befehl  <Menu>Daten importieren</Menu>  in der Werkzeugleiste von  <Placeholder>fullproduct</Placeholder> oder &#252;ber das  <Menu> Menu: [Daten - Daten importieren] </Menu>  eingef&#252;gt.

Die Dateien werden in eine rmDATA Geodatabase importiert, diese wird danach in das aktuelle Geodokument eingebunden.

Nach Befehlsaufruf wird im Register  <Border>Typauswahl</Border>  die Option  <Border>Daten aus NAS-Dateien importieren</Border>  gew&#228;hlt.<Image>\img\grafik\fachschaleliegenschaftskatasterde\ImportNASTypauswahl.png$ImportNASTypauswahl</Image>

Danach werden im Register  <Border>Dateiauswahl</Border>  die zu importierenden NAS-Dateien gew&#228;hlt. Es k&#246;nnen sowohl XML-Dateien als auch gezippte Arichve (*.zip) ausgew&#228;hlt werden.<Image>\img\grafik\fachschaleliegenschaftskatasterde\ImportNASDateiauswahl.png$ImportNASDateiauswahl</Image>

Im Register  <Border>Geodatabase-Auswahl</Border>  wird festgelegt, in welche Geodatabase die Dateien importiert werden sollen.<Image>\img\grafik\fachschaleliegenschaftskatasterde\ImportNASGeoDBAuswahl.png$ImportNASGeoDBAuswahl</Image>

In den  <Border>Einstellungen</Border>  wird festgelegt

- ob bestehende Daten aus der Datenbank gel&#246;scht werden sollen. Diese Option wirkt sich nur aus, wenn in eine bestehende Datenbank importiert wird. 


<Warning>
 Bei Projekten in einer Enterprise GeoDB, welche viele Daten enthalten, kann das L&#246;schen der bestehenden Daten lange dauern, wodurch es zu einem Timeout kommen kann. In so einem Fall kann &#252;ber die rmdata_konfiguration.xml in der Sektion &quot;GeoDesktop-EnterpriseGeoDBSettings&quot; mit dem Eintrag &quot;CustomCommandTimeout&quot; der Timeout in Sekunden ver&#228;ndert werden. 

</Warning>
- aus welchem Bundesland die NAS-Dateien stammen

- welche Modellarten &#252;bernommen werden sollen

- ob Daten aus einem bestimmten Koordinatensystem &#252;bernommen werden sollen (wird nur angeboten, wenn in den Basisdaten unterschiedliche Koordinatensysteme definiert sind). 


<Warning>
Das hier gew&#228;hlte Koordinatensystem muss mit dem unter &quot;Dateiauswahl&quot; eingestellten Koordinatensystem zusammenpassen!

</Warning>
- welche Themen f&#252;r den Import herangezogen werden sollen. F&#252;r einen Standard-GIS-Import werden die Themen &quot;Bauwerke&quot;, &quot;Eigent&#252;merdaten&quot;, &quot;Flurst&#252;cke, Lage Punkte&quot;, &quot;Geb&#228;udedaten&quot; sowie &quot;Nutzungsarten&quot; empfohlen.<Image>\img\grafik\fachschaleliegenschaftskatasterde\ImportNASEinstellungen.png$ImportNASEinstellungen</Image>

Nach  <Border>Fertigstellen</Border>  werden die Daten importiert, ein Transferstatus wird angezeigt.<Image>\img\grafik\fachschaleliegenschaftskatasterde\ImportNASTransferstatus.png$ImportNASTransferstatus</Image>

In den  <Border>Einstellungen</Border>  werden nocheinnmal die importierten Objektklassen angezeigt:<Image>\img\grafik\fachschaleliegenschaftskatasterde\ImportNASEinstellungen2.png$ImportNASEinstellungen2</Image>

In den  <Border>erweiterten Einstellungen</Border>  kann die Darstellung und der Defaultma&#223;stab festgelegt werden:<Image>\img\grafik\fachschaleliegenschaftskatasterde\ImportNASEinstellungenerweitert.png$ImportNASEinstellungenerweitert</Image>

Der Importvorgang ist nun beendet und die Geodatabase wird ins Projekt eingebunden.
