## Assistent f&#252;r das Erstellen einer Konvertierungstabelle f&#252;r den Import von Autodesk DWG oder DXF Dateien

F&#252;r den Import von Autodesk DWG/DXF Dateien in eine rmDATA Geodatabase steht ein Mapping-Assistent zur Verf&#252;gung, sodass die Objekte der Autodesk-Dateien in Objekte der Geodatabase &#252;bernommen werden k&#246;nnen.<Image>\img\grafik\daten\mappingdwg1.png$mappingdwg1</Image>

Im ersten Schritt wird der Inhalt der Import-Dateien analysiert.
Danach werden die zuordenbaren Objektklassen in Registern angezeigt.
F&#252;r jede Geodatabase-Objektklasse werden alle Layer einer AutoCAD-Gruppe (Insert, Point, Text, ...) aufgelistet, und es kann jedem Layer ein Objekttyp der Geodatabase zugeordnet werden.
Liegen Bl&#246;cke auf unterschiedlichen Layern, wird f&#252;r jeden Block eine Zeile mit einer Bedingung angelegt. Als Bedingungen k&#246;nnen auch Eigenschaften wie Farbe, Textstil, Texth&#246;he oder Linientyp definiert werden.
Im Dialog k&#246;nnen einzelne Zeilen dupliziert und danach ge&#228;ndert werden:  <Border>+</Border> ,  <Border>x</Border> 
In der Spalte &quot;Objekttyp der GeoDB&quot; wird vom Anwender der gew&#252;nschte Objekttyp der Geodatabase eingestellt. 
Weiters k&#246;nnen dem Objekttyp Attribute (entweder aus dem AutoCAD-Objekt, oder fixe Attributwerte) zugewiesen werden.
Jedes AutoCAD-Objekt kann nur einmal &#252;bernommen werden. Bei Doppeldefinitionen gilt die Reihenfolge der Register und die Reihenfolge der Zeilen.

<Success>
Nach  <Border>Fertigstellen</Border>  kann das erstellte Mapping in einer Datei gespeichert werden.
</Success>
