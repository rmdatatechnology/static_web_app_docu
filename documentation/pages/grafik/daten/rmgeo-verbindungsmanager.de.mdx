## Herstellen einer Verbindung zu rmGEO4
 <Menu>Daten/ rmGEO-Verbindungsmanager</Menu> 
<GeoDesigner> <Menu> Cmd: [RmgeoConnectionManager] </Menu> 

 <Placeholder>geodgm</Placeholder> </GeoDesigner>
&#220;ber den Verbindungsmanager wird eine neue Verbindung zu einer rmGEO-Datenquelle hergestellt, die Einstellungen einer bestehenden Verbindung ge&#228;ndert oder eine Verbindung getrennt.

### Automatischer Abgleich

<GeoMapper>

- Rufen Sie den Befehl auf, um den Verbindungsmanager zu starten

- Ist noch keine rmGEO-Datenquelle verbunden:

- Der Verbindungsmanager wird ge&#246;ffnet

- W&#228;hlen Sie den Pfad zur neuen rmGEO-Datenquelle

- W&#228;hlen Sie weitere Optionen

- Dr&#252;cken Sie <Border>OK</Border> , um die Einstellungen zu &#252;bernehmen.

- Es erfolgt eine Abfrage, ob das Projekt gespeichert werden soll. 

- Es werden alle Daten abgeglichen (Siehe 
[Konsistenzpr&#252;fung](../daten/rmgeo-konfliktaufloesung)
) und die Verbindung hergestellt.



- Ist bereits eine Verbindung zu einer rmGEO-Datenquelle vorhanden:

- Der Verbindungsmanager wird ge&#246;ffnet

- &#196;ndern Sie nach Bedarf die gew&#252;nschten Einstellungen

- Dr&#252;cken Sie <Border>OK</Border> , um die Einstellungen zu &#252;bernehmen.

- Je nach erfolgten &#196;nderungen der Einstellungen werden alle Daten 
[abgeglichen](../daten/rmgeo-konfliktaufloesung)
.






Im Verbindungsmanager wird folgendes festgelegt:


- <Bold> Pfad </Bold> zum neuen rmGEO-Projekt

- <Bold> Priorit&#228;t f&#252;r Punkttyp </Bold> : Ist die Option <Italic> Grafik </Italic>  gesetzt, werden bei Differenzen im Punkttyp immer die Punkttypen aus der <Placeholder>product</Placeholder> Zeichnung verwendet und in rmGEO automatisch &#252;berschrieben. Ist die Option auf rmGEO gesetzt, erfolgt der Abgleich gem&#228;&#223; den Einstellungen beim Vorrang f&#252;r bestehende Punkte. Alle anderen Attribute werden IMMER gem&#228;&#223; den Vorrangeinstellungen behandelt.

- <Bold> Standardpriorit&#228;t f&#252;r Konfliktdialog </Bold>  f&#252;r bestehende Punkte der <Placeholder>product</Placeholder> Datei: Werden beim Abgleich Inkonsistenzen gefunden, werden diese in einer Liste ausgegeben. In dieser Liste w&#228;hlt der Anwender f&#252;r jeden Punkt aus, ob dieser aus rmGEO oder aus <Placeholder>product</Placeholder> &#252;bernommen wird. Die Einstellung gibt an, ob rmGEO oder <Placeholder>product</Placeholder> defaultm&#228;&#223;ig als Vorrang eingestellt ist. 

- <Border>Attribute</Border>  Steuert den Abgleich der Attribute. Grunds&#228;tzlich werden automatisch alle Attribute gleichen Namens abgeglichen, welche in rmGEO definiert sind und in <Placeholder>product</Placeholder> als Klassenattribute vorliegen. Attribute, die in <Placeholder>product</Placeholder> nicht als Klassenattribute vorliegen, k&#246;nnen vom Anwender bei Bedarf gemappt werden. Gemappte Attribute werden, sofern sie bei einem Objekt nicht bereits vorkonfiguriert sind, als freie Attribute hinzugef&#252;gt. 

- Einen Sonderfall stellt das Attribut Klassifizierung dar. Wenn Punkte in der Version <Italic> neu </Italic>  von <Placeholder>product</Placeholder> nach rmGEO &#252;bertragen werden, wird bei den Punkten in rmGEO das Attribut <Bold> Klassifizierung </Bold>  gesetzt. Zus&#228;tzlich wird auch das rmGEO-Attribut <Bold> KlassifizierungBer </Bold>  mit dem Wert aus der Version <Italic> berichtigt </Italic>  gesetzt.



- <Border>Indikatoren</Border> Hier k&#246;nnen Mappings von rmGEO-Festcodes zu Indikatoren konfiguriert werden. Je nach Indikator wird f&#252;r jeden Punkt der &#196;nderungsschutz unterschiedlich gesetzt.




<Success>
Ist in  <Placeholder>product</Placeholder> die Versionierung aktiv, dann werden Punkte jeweils in ihrer letztg&#252;ltigen Version nach rmGEO &#252;bertragen.
</Success>

<Success>
Neue Punkte in rmGEO werden in die in  <Placeholder>product</Placeholder> gerade aktive Version eingef&#252;gt, sofern in rmGEO keine Version gesetzt ist.
</Success>

<Success>
Wenn Sie einen Punkt in GeoMapper umbennenen, wird beim Abgleich mit rmGEO auch der entsprechende GNSS-Punkt im Stand 989 mit umbenannt.
</Success>

- Rufen Sie den Befehl auf, un den Verbindungsmanager zu starten

- W&#228;hlen Sie <Bold> Datei </Bold>  f&#252;r eine dateibasierte rmGEO-Datenquelle (RMG-Datei) oder <Bold> Zentrale Datenbank </Bold>  um ein rmGEO-Projekt aus einer Oracle- bzw. SQL-Server-Datenbank anzubinden.






Weiters werden folgenden Einstellungen festgelegt:


- <Bold> Attribut-Mapping </Bold> : Diese Einstellung ist optional. Ist aber eine Datei angegeben, werden die darin aufgelisteten rmGEO-Attribute mit Attributen zu <Placeholder>product</Placeholder> Punkten abgeglichen. Die Datei ist eine normale Textdatei bestehend aus 2 Spalten, die durch Leerzeichen bzw. Tabulatoren getrennt sind. In der ersten Spalte steht das rmGEO-Attribut, in der zweiten Spalte das <Placeholder>product</Placeholder> Punktattribut.


<Warning>
 <Placeholder>product</Placeholder> listet die verf&#252;gbaren Mapping-Dateien aus einer Reihe von standardisierten Ordnern, sodass Sie sie leichter finden k&#246;nnen. Sollten Sie eine Geodatenbank aus einer &#228;lteren Programmversion verwenden, wo ein Attribut-Mapping aus einem beliebigen Ordner verwendet werden konnte, dann weist 
 <Placeholder>product</Placeholder> Sie auf diesen Umstand hin, sobald Sie erneut den Verbindungsdialog &#246;ffnen. Dies dient ausschlie&#223;lich einer standardisierten Arbeitsweise; der Abgleich selbst funktioniert auch mit Mapping-Dateien, die aus beliebigen Ordnern stammen.


<Image>\img\grafik\daten\GeoMappingWarning.png$GeoMappingWarning</Image>
</Warning>
- <Bold> Leere Attributwerte ignorieren </Bold> : Im Abgleichsdialog gibt es Einstellungen f&#252;r den Vorrang (Attribut aus rmGEO gewinnt oder Attribut aus <Placeholder>product</Placeholder> gewinnt). Ist einer der beiden Attribute leer und diese Option aktiv, wird standardm&#228;&#223;ig immer das bef&#252;llte Attribut verwendet.



<Success>
Folgende Attribute in rmGEO werden immer abgeglichen und m&#252;ssen im Attribut-Mapping nicht ber&#252;cksichtigt werden: Koordinaten (Y, X und H), Pkttyp, Festcode, Messcode, Klassifizierung bzw. KlassifizierungBer, Vhw/GFN, Status  <Italic> sowie alle Attribute, die in rmGEO und  <Placeholder>product</Placeholder> gleich hei&#223;en und den gleichen Datentyp haben. </Italic> 
</Success>


- <Bold> Abgleich beim Schlie&#223;en aufrufen </Bold> : Ruft den Abgleich automatisch beim Schlie&#223;en eines Projektes auf.

- <Bold> Filter: </Bold> 
- <Bold> Geometrischer Filter: </Bold>  Schr&#228;nken Sie den Bereich durch Klick auf <Border>...</Border>  r&#228;umlich ein. Dabei k&#246;nnen Sie ein Rechteck aufziehen oder in den Optionen zur polygonalen Auswahl wechseln.

- <Bold> Nur sichtbare Punkte: </Bold>  Nur in der Grafik sichtbare Punkte werden abgeglichen






###  Trennen einer bestehenden Verbindung

- Rufen Sie den Befehl auf, um den Verbindungsmanager zu starten

- Klicken Sie den Button<Image>\img\grafik\daten\ConnDelete.png$ConnDelete</Image> neben dem Verbindungsnamen und best&#228;tigen Sie den Dialog mit <Border>OK</Border> .

- Vor dem Beenden der Datenquelle muss das Projekt gespeichert werden; eine entsprechende Abfrage wird aufgerufen

- W&#228;hlen Sie, ob das Projekt gespeichert werden soll

- Wird das Projekt gespeichert, wird die Verbindung zur rmGEO-Datenquelle beendet

- Wird das Projekt nicht gespeichert, bleibt die Verbindung zur rmGEO-Datenquelle aufrecht






<Success>
Sollen &#196;nderungen nicht nach rmGEO &#252;bertragen werden, muss das Projekt ohne Speichern beendet werden.
</Success>
