## Importieren von Daten in eine rmDATA Geodatabase

Mit dem Befehl  <Menu>Daten importieren</Menu>  werden externe Daten in eine rmDATA Geodatabase importiert.

Je nach aktivierter Fachschale stehen unterschiedliche Importe zur Verf&#252;gung:

- Viewingmodus: <Menu> Menu: [Daten - Daten importieren] </Menu> . Alle Daten werden in eine bestehende oder eine neue rmDATA Geodatabase importiert. Nach dem Import wird die Geodatabase in das aktuelle Geodokument eingebunden.

- Importe der 
[Fachschale DKM](../fachschalegrundstueckskatasterundadressenat/datenimportieren)

- [Punkte aus ASCII Daten importieren](../daten/importascii)

- [Sachdaten aus ASCII Daten importieren](../daten/importsachdaten)

- [Daten aus Autdesk DWG Dateien importieren](../daten/importautodeskdwg)

- [Daten aus Autdesk DXF Dateien importieren](../daten/importautodeskdwg)

- [Daten aus Post-Anhang-A DXF Dateien importieren](../daten/importpostanhangadxf)

- [Daten aus Esri Shape Dateien importieren](../daten/importesrishape)

- [Daten aus IGLIS importieren](../daten/importiglis)

- [Daten aus rmDATA XML Dateien importieren](../daten/importrmdataxml)







- Editiermodus: <Menu> Menu: [Datenbank - Daten importieren] </Menu>  (nur mit aktivierter Fachschale &quot;Editieren&quot;). Alle Daten werden in die gerade zum editieren ge&#246;ffnete rmDATA Geodatabase importiert.

## Default-Importverzeichnis
In der Datei  <Menu>rmdata_konfiguration.xml</Menu>  im Firmenverzeichnis oder im Benutzerverzeichnis kann ein Default-Verzeichnis festgelegt werden:

 <Menu>  section name="GeoDesktop-Folders"</Menu> 
 <Menu>    entry name="DefaultImportDirectory"C:\Temp/entry</Menu> 
 <Menu>  /section</Menu> 



<Success>
Mit  <Border>Vorlage speichern</Border>  speichern Sie die Einstellungen in einer Vorlage mit einem selbst gew&#228;hlten Namen. Beim n&#228;chsten Start des Importmanagers brauchen Sie dann nur Ihren Import w&#228;hlen und haben alle Einstellungen passend gesetzt.
</Success>
