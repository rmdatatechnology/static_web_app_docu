## Cr&#233;ation d&#39;une connexion avec rmGEO4

Par le gestionnaire de connexions on peut cr&#233;&#233;r une nouvelle connexion &#224; une source de donn&#233;es rmGEO, modifier les param&#232;tres d&#39;une connexion existante ou bien couper une connexion.

### Comparaison automatique<Image>\img\grafik\daten\rmGEO.Verbindungsmanager.png$rmGEO.Verbindungsmanager</Image>

- Appelez la commande pour d&#233;marrer le gestionnaire de connexions.

- Si aucune source de donn&#233;es rmGEO n&#39;est encore connect&#233;e:

- Le gestionnaire de connexions s&#39;ouvre.

- S&#233;lectionnez le chemin pour la nouvelle source de donn&#233;es rmGEO.

- Choisissez les autres options.

- Appuyez sur <Border>OK</Border> , pour accepter les param&#232;tres.

- Un message vous demande si le projet doit &#234;tre enregistr&#233;. 

- Toutes les donn&#233;es sont compar&#233;es (voir 
[V&#233;rification de consistance](../daten/rmgeo-konfliktaufloesung)
) et la connexion est cr&#233;&#233;e.



- Si une connexion &#224; une source de donn&#233;es rmGEO est d&#233;j&#224; pr&#233;sente:

- Le gestionnaire de connexions s&#39;ouvre.

- Modifiez si n&#233;cessaire les param&#232;tres d&#233;sir&#233;s.

- Appuyez sur <Border>OK</Border> , pour accepter les param&#232;tres.

- Selon les modifications ex&#233;cut&#233;es, toutes les donn&#233;es sont 
[compar&#233;es](../daten/rmgeo-konfliktaufloesung)
.






Dans le gestionnaire de connexions les param&#232;tres suivants sont d&#233;finis:


- <Bold> Chemin </Bold>  vers le nouveau projet rmGEO.

- <Bold> Priorit&#233; pour le type de point </Bold> : si l&#39;option <Italic> Graphique </Italic>  est activ&#233;e, lors des diff&#233;rences de type de point, les types de point depuis le dessin <Placeholder>product</Placeholder> sont toujours reprises et dans rmGEO &#233;cras&#233;s automatiquement. Si l&#39;option <Italic> rmGEO </Italic>  est activ&#233;e, la comparaison est effectu&#233;e selon les param&#232;tres o&#249; la priorit&#233; est donn&#233;e aux points existants. Tous les autres attributs sont TOUJOURS trait&#233;s selon les d&#233;finitions de priorit&#233;.

- <Bold> Priorit&#233; standard pour le dialogue des conflits </Bold>  pour les points existants du fichier <Placeholder>product</Placeholder>  lors de la comparaison, les inconsistences sont affich&#233;es dans une liste. Dans cette liste l&#39;utilisateur peut choisir pour chaque point, si reprendre-le depuis rmGEO ou depuis <Placeholder>product</Placeholder>  Le param&#232;tre d&#233;finit si la priorit&#232; par d&#233;faut est donn&#233;e &#224; rmGEO ou &#224; <Placeholder>product</Placeholder>  

- <Border>Attributs</Border>  g&#232;re la comparaison des attributs. En principe tous les attributs avec le m&#234;me nom sont compar&#233;s automatiquement, lesquels sont d&#233;finis dans rmGEO et dans <Placeholder>product</Placeholder> existent comme attributs de classe. Les attributs, qui dans <Placeholder>product</Placeholder> ne sont pas d&#233;finis comme attributs de classe, si n&#233;cessaire peuvent &#234;tre mapp&#233;s par l&#39;utilisateur. Les attributs mapp&#233;s, &#224; condition qu&#39;ils ne soient pas pr&#233;configur&#233;s pour un objet, sont ins&#233;r&#233;s comme attributs libres. 

- L&#39;attribut de classification repr&#233;sente un cas particulier. Quand des points en version <Italic> nouveau </Italic>  sont transf&#233;r&#233;s depuis <Placeholder>product</Placeholder> vers rmGEO, dans rmGEO l&#39;attribut <Bold> Classification </Bold>  est d&#233;fini pour les points. En outre il est d&#233;fini l&#39;attribut rmGEO <Bold> ClassificationCalc </Bold>  avec la valeur depuis la Version <Italic> corrig&#233; </Italic> .



- <Border>Indicateurs</Border>  Ici on peut configurer le mappage depuis les codes de points fixes de rmGEO vers les indicateurs. En fonction de l&#39;indicateur, pour chaque point est d&#233;finie une protection diff&#233;rente contre les modifications.




<Success>
Si dans  <Placeholder>product</Placeholder> le versionnement est actif, alors tous les points, chacun dans sa propre derni&#232;re version valable, sont transf&#233;r&#233;s dans rmGEO.
</Success>

<Success>
Les nouvelles points dans rmGEO sont ins&#233;r&#233;s dans  <Placeholder>product</Placeholder> dans la version active, &#224; condition que dans rmGEO aucune version ne soit pas d&#233;finie.
</Success>


### Comparaison manuelle<Image>\img\grafik\daten\rmGEO.VerbindungsmanagerManuell.png$rmGEO.VerbindungsmanagerManuell</Image>

- Appelez la commande pour d&#233;marrer le gestionnaire de connexions.

- S&#233;lectionnez <Bold> Fichier </Bold>  pour une source de donn&#233;es rmGEO (RMG-Datei) bas&#233;e sur ficher ou <Bold> Base de donn&#233;es centrale </Bold>  pour connecter un projet rmGEO depuis une base de donn&#233;es Oracle resp. SQL-Server.






Autres param&#232;tres &#224; d&#233;finir:


- <Bold> Mappage-Attribut </Bold> : D&#233;finition optionelle. Si par contre on donne un fichier, les attributs rmGEO ici &#233;num&#233;r&#233;s sont compar&#233;s avec les attributs de point de <Placeholder>product</Placeholder>  Il s&#39;agit d&#39;un ficher texte normal avec 2 colonnes, s&#233;par&#233;es par Espaces ou Tabulateurs. Dans la premi&#232;re colonne on trouve l&#39;attribut rmGEO, dans la deuxi&#232;me colonne l&#39;attribut de point de <Placeholder>product</Placeholder> 
- <Bold> Ignorer attributs vides </Bold> : Dans le dialogue de comparaison il y a les param&#232;tres pour la priorit&#233; (l&#39;attribut depuis rmGEO gagne ou l&#39;attribut depuis <Placeholder>product</Placeholder> gagne). Quand un des attributs est vide et que cette option est active, par d&#233;faut, l&#39;attribut rempli est celui qui est toujours utilis&#233;.



<Success>
Les attributs rmGEO suivants sont toujours compar&#233;s et donc ne doivent pas &#234;tre consid&#233;r&#233;s dans le Mappage-Attribut: Coordonn&#233;es (Y, X et H), Type de point, Code point fixe, Code mesure, Classification resp. ClassificationCalc, VHW/GFN, Etat.
</Success>


### Couper une connexion existante

- Appelez la commande pour d&#233;marrer le gestionnaire de connexions.

- Cliquez sur le bouton<Image>\img\grafik\daten\ConnDelete.png$ConnDelete</Image> &#224; cot&#233; du nom de connexion et confirmez le dialogue avec <Border>OK</Border> .

- Avant de terminer la source de donn&#233;es il faut sauvegarder le projet; une question appropri&#233;e est affich&#233;e.

- Choisissez si vous d&#233;sirez enregistrer le projet.

- Le projet est enregistr&#233;, la connexion &#224; la source de donn&#233;es rmGEO est coup&#233;e.

- Le projet n&#39;est pas enregistr&#233;, la connexion &#224; la source de donn&#233;es rmGEO est maintenue.






<Success>
Si vous ne d&#233;sirez pas reprendre les modifications dans rmGEO, le projet doit se terminer sans l&#39;enregistrer.
</Success>
