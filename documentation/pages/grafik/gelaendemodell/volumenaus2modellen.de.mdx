## Berechnen Sie das Differenzvolumen aus 2 Modellen.
<GeoDesigner> <Menu> Menu: [ ^^dtmRibbonTitle^^ / Volumen / Volumen aus 2 Modellen] </Menu> </GeoDesigner><GeoMapper> <Menu> Menu: [Geländemodell / Volumen aus 2 Modellen] </Menu> </GeoMapper>


1. Rufen Sie den Befehl auf.

2. W&#228;hlen Sie im Dialog das zweite Modell aus. Als Modell 1 wird standardm&#228;&#223;ig das Aktive Modell eingetragen.

3.  Geben Sie <Bold> optional </Bold>  die Dicke der Humusschicht an, die Sie f&#252;r die Berechnung ber&#252;cksichtigen wollen. Falls die keine Humusschicht ber&#252;cksichtigen m&#246;chten, deaktivieren Sie einfach die entsprechende Option im Dialog.

4. Klicken Sie auf <Border>Berechnen</Border> . Die Volumina werden berechnet und im Protokoll ausgegeben.

 <U>  <Bold> Weitere Einstellungen im Dialog: </Bold>  </U> 

- <Bold> Prismen zeichnen: </Bold>  In der Grafik werden die einzelnen Prismen dargestellt und ihr Volumen angeschrieben

- <Bold> Verschnittlinie zeichnen: </Bold> Falls sich die Ebenen schneiden, wird die Verschnittlinie in die Grafik eingezeichnet.

- <Bold> REB Datei erzeugen: </Bold>  Die Volumen werden in eine ASCII-Datei im REB-Format geschrieben.

- <Bold> CSV Datei erzeugen: </Bold> Die Volumen werden in eine ASCII-Datei geschrieben. Alle Inhalte sind mit einem Trennzeichen getrennt. Die Datei kann mit <Italic> MS Excel </Italic>  ge&#246;ffnet werden.



<Success>
Das Volumen der Humusschichte wird als Produkt aus der angegebenen Humusdicke und der Oberfl&#228;che des berechneten Modells ermittelt. F&#252;r die Volumenberechnung des Modells wird die Humusdicke von jedem Punkt des Modells abgezogen. Der eingestellte Humusabtrag, der einer Absenkung des Modells entspricht, wird im Protokoll gesondert ausgegeben.
</Success>

<Success>
Im Protokoll werden Verkn&#252;pfungen auf die Ausgabedateien (REB und/oder CSV-Datei) erzeugt. Damit Sie diese Dateien direkt aus dem Protokoll &#246;ffnen k&#246;nnen, ist es notwendig, dass ein Programm zum &#214;ffnen mit der jeweiligen Dateiendung verkn&#252;pft ist. N&#228;here Informationen entnehmen Sie bitte der Dokumentation des Betriebssystems ( <Italic> &quot;Zuordnen einer Datei zu einem Programm&quot; </Italic> ).
</Success>

<Warning>
Falls die Verschnittslinien zwischen zwei Modellen nicht mit ausreichender Genauigkeit ermittelt werden k&#246;nnen, wird dies mit einem entsprechenden Eintrag in Protokoll dokumentiert, die Berechnung wird unterbrochen und kein Ergebnis wird ausgegeben. In so einem Fall haben Sie zwei M&#246;glichkeiten zur Verf&#252;gung:
1.	Die Gel&#228;ndeoberfl&#228;chen sollen genauer modelliert werden (durch Einf&#252;gen zus&#228;tzliche Punkte) oder
2.	Die H&#246;henbereinigungstoleranz soll herabgesenkt werden (Default &#8211 Wert : 5 cm).
</Warning>

<Success>
Sie k&#246;nnen das Volumen &#252;ber eine begrenze Fl&#228;che berechnen, indem Sie eine Volumengrenze  <Menu>erzeugen </Menu> oder mittels der Option  <Menu>Alles rechnen</Menu>  das Volumen innerhalb der  Modellgrenze berechnen. Die Auswahl dieser Optionen ist direkt &#252;ber die Optionsleiste m&#246;glich. Siehe auch [Volumengrenze zeichnen](../gelaendemodell/volumengrenzezeichnen)
</Success>
