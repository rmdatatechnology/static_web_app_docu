## Op&#233;ration &quot;Variation d&#39;altitude&quot;
Les donn&#233;es d&#39;origine du mod&#232;le sont modifi&#233;es directement ou utilis&#233;es comme copie modifi&#233;e pour un nouveau mod&#232;le.

Options param&#232;trables:

- <Bold> Mod&#232;le d&#39;origine </Bold> : Mod&#232;le dont on veut utiliser les donn&#233;es (il doit contenir des donn&#233;es attribu&#233;es, mais il ne doit pas &#234;tre calcul&#233;).

- <Bold> Altitude </Bold> : La variation d&#39;altitude d&#233;sir&#233;e en m&#232;tres.

- <Bold> R&#233;sultat </Bold> :

- <Bold> Modifier mod&#232;le d&#39;origine </Bold> : Toutes les donn&#233;es d&#39;origine attibu&#233;es sont g&#233;om&#233;triquement modifi&#233;es dans l&#39;altitude (points, nuages de points, polylignes comme contour de mod&#232;le ou lignes de ruptures).

- <Bold> Nouveau mod&#232;le </Bold> : Toutes les donn&#233;es d&#39;origine sont dupliqu&#233;es, l&#39;altitude est modifi&#233;e et le tout est affect&#233; au nouveau mod&#232;le (points, nuages de points, polylignes comme contour de mod&#232;le ou lignes de ruptures)



- <Bold> Nom du nouveau mod&#232;le </Bold>  (il doit &#234;tre univoque)

- <Bold> Types d&#39;objets </Bold> :

- <Bold> Mod&#232;le de terrain </Bold> 
- <Bold> Type de point </Bold> 
- <Bold> Type de nuages de points </Bold> 
- <Bold> Name du nuages de points </Bold>  (il doit &#234;tre univoque)

- <Bold> Contour de mod&#232;le </Bold> 
- <Bold> Lignes de rupture </Bold> 
- <Bold> Lignes de rupture cassure </Bold> 




## Operation &quot;Mod&#232;le par diff&#233;rence&quot;
Depuis le maillage de deux mod&#232;les existants, un mod&#232;le par diff&#233;rence est calcul&#233;, dont les altitudes des noeuds sont calcul&#233;es depuis la diff&#233;rence d&#39;altitude.

Options param&#232;trables:

- <Bold> Mod&#232;le d&#39;origine </Bold> : Le mod&#232;le depuis lequel on veut soustraire le deuxi&#232;me mod&#232;le (il doit &#234;tre calcul&#233;).

- <Bold> Mod&#232;le &#224; soustraire </Bold> : Le mod&#232;le &#224; soustraire pour cr&#233;er le mod&#232;le par diff&#233;rence (il doit &#234;tre calcul&#233;).

- <Bold> Nom du nouveau mod&#232;le </Bold> : (il doit &#234;tre univoque).



Le maillage du mod&#232;le par diff&#233;rence peut aussi export&#233; ou utilis&#233; pour une repr&#233;sentation th&#233;matique.
<Success>
Le mod&#232;le par diff&#233;rence r&#233;sultant ne contient aucune donn&#233;es d&#39;origine et donc ne peut pas &#234;tre modifi&#233; ou calcul&#233; &#224; nouveau.
</Success>
