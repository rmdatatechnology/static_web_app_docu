### Einstellungen f&#252;r die Punktbezeichnung
Punktnamen in  <Placeholder>fullproduct</Placeholder> bestehen aus 

-  Region / KG / Nummerierungsbezirk

-  Punktbezeichnung

-  Subname



Die Region und der Subname k&#246;nnen in der Grafik bzw. bei Punkteingaben ausgeblendet werden.

- <Bold> Trennzeichen: </Bold>  Trennzeichen zwischen den Teilen des Punktnamens





- <Bold> F&#252;hrende Nullen ausblenden: </Bold>  Punktbezeichnungen und Subnamen in <Placeholder>fullproduct</Placeholder> werden f&#252;r alle grafischen Ausgaben ohne f&#252;hrende 0 ausgegeben.





- <Bold> Punktnummern automatisch vergeben: </Bold>  Beim Einf&#252;gen von Punkten wird die n&#228;chste freie Punktnummer automatisch dem neuen Punkt zugeordnet. Ansonsten wird der Dialog zur Eingabe von Sachdaten angezeigt, die Punktnummer muss dann h&#228;ndisch angegeben werden. Die n&#228;chste freie Punktnummer ist in der Statuszeile sichtbar bzw. kann dort ge&#228;ndert werden.




### Punktkollision
Gibt den Vorschlagswert bei der Punktkollision an.

### Versionierung

- <Bold> Koordinaten in allen Versionen gleich: </Bold>  Wird ein versionierter Punkt in einer neueren Version verschoben, dann werden die Koordinaten - wenn die Option nicht angehakt ist - nur in dieser Version ver&#228;ndert. Ist sie angehakt, dann wird der Punkt in allen Versionen verschoben.



<Success>
Um sicher zu stellen, dass auch in bereits bestehenden Projekten keine Koordinatendifferenzen bestehen, kann die Option Koordinaten in allen Versionen gleich beim &#214;ffnen von Projekten gepr&#252;ft und automatisch aktiviert werden. Dazu ist in der Datei rmdata_konfiguration.xml der folgende Eintrag zu erg&#228;nzen: 

```
&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt; 
&lt;profile&gt;
   &lt;section name=&quot;UserSettings&quot;&gt;
     &lt;entry name=&quot;LinkVersionedKoordinatesDefault&quot;&gt;True&lt;/entry&gt;
   &lt;/section&gt;
 [...]
 &lt;/profile&gt; 
```

Wenn diese Einstellung gesetzt wurde, wird beim &#214;ffnen einer versionierten Datei gepr&#252;ft, ob die Option Koordinaten in allen Versionen gleich aktiv ist. Wenn das nicht der Fall ist, wird der Einstellungsdialog ge&#246;ffnet und die Option aktiviert. Wenn Koordinatendifferenzen vorhanden sind, kann die ge&#228;nderte Einstellung nicht &#252;bernommen werden. Die Koordinatendifferenzen m&#252;ssen vorab z.B. mit dem Befehl Versions&#252;bergreifender Punktvergleich, behoben werden. 
<GeoMapper>Die rmdata_konfiguration.xml kann im Programmdaten-Verzeichnis (%programdata%\rmdata\GeoMapperSE), im Benutzerdatenverzeichnis (%appdata%\rmdata\GeoMapperSE) oder im Firmenverzeichnis liegen.
</GeoMapper>
