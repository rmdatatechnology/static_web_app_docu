## Param&#232;tres pour le Smart Picks et l&#39;affichage dans la Vue-3D.
<GeoMapper>
<Success>
Ces param&#232;tres sont visibles, seulement si l&#39;application &quot;G&#233;om&#232;tre virtuel&quot; est active.
</Success><Image>\img\grafik\datei\GeoMapper.programmeinstellungen_virtuellervermesser.png$GeoMapper.programmeinstellungen_virtuellervermesser</Image>


- <Bold> Vue-3D </Bold> 
- <Bold> Plage de visibilit&#233; maximale: </Bold>  Profondeur jusqu&#39;&#224; laquelle la vue en 3D est affich&#233;e. 


<Success>
Pour gagner en performance, limitez la plage de visibilit&#233; en cas de grosses donn&#233;es Scans.

</Success>






- <Bold> Param&#232;tres d&#39;accrochage </Bold> 
- <Bold> Rayon d&#39;accrochage (en Pixel): </Bold>  Il d&#233;finit la dimension de la zone d&#39;accrochage autour de l&#39;axe optique.





<Success>
Plus vous zoomez dans la vue et plus la zone d&#39;accrochage se reduit autour de l&#39;axe optique.
</Success>



- <Bold> SmartPick: </Bold>  (Voir aussi 
[Utilisation du Smart Picks](../virtuellervermesser/smartpick)
)

- <Bold> Synchrone / Fix&#233;: </Bold>  Si on tourne dans la Vue-3D, la vue tourne aussi dans le SmartPick-Manager. Si on fixe la vue du SmartPick-Manager, ceci peut &#234;tre tourn&#233; en intervalles de 45 d&#233;gr&#233;es ind&#233;pendamment de la vue en 3D. 

- <Bold> Dimension standard (en m&#232;tres): </Bold> Dimension de l&#39;extrait standard affich&#233; dans le SmartPick-Manager. La dimension de l&#39;extrait peut ensuite encore &#234;tre modifi&#233;e.

- <Bold> Point le plus proche: </Bold> 
- <Bold> Direction de vis&#233;e (Standard): </Bold> Le point le plus proche &#224; l&#39;axe visuel est piqu&#233;.

- <Bold> Axe-Z : </Bold>  Le point avec la coordonn&#233;e altim&#233;trique la plus petite est piqu&#233;.



- <Bold> Point le plus haut: </Bold> 
- <Bold> Axe-Z (Standard) : </Bold> Le point avec la coordonn&#233;e altim&#233;trique la plus grande, dans le rayon d&#39;accrochage, est piqu&#233;.

- <Bold> Direction de vis&#233;e: </Bold> Le point le plus haut le long la direction de vis&#233;e est piqu&#233;.



- <Bold> Point le plus bas: </Bold> 
- <Bold> Axe-Z (Standard) : </Bold> Le point avec la coordonn&#233;e altim&#233;trique la plus petite, est piqu&#233;.

- <Bold> Direction de vis&#233;e: </Bold> Le point le plus bas le long la direction de vis&#233;e est piqu&#233;.



- <Bold> Point de base: </Bold> 
- <Bold> Axe-Z (Standard) : </Bold> D&#233;pendant du mod&#232;le de terrain, autour de l&#39;objet scann&#233; le point de base est piqu&#233;.

- <Bold> Direction de vis&#233;e: </Bold> Le point d&#39;intersection entre le mod&#232;le de terrain et la direction de vis&#233;e, est piqu&#233;. 



- <Bold> Rayon de base: </Bold> Il d&#233;finit le rayon utilis&#233; pour d&#233;terminer l&#39;altitude moyenne de la base pour l&#39;accrochage du &#8222Point de base&#8220. 






</GeoMapper>
