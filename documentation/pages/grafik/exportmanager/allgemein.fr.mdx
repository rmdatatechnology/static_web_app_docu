## Avec l&#39;Export-Manager vous pouvez exporter les donn&#233;es des vos fichiers en diff&#233;rents formats.
<GeoMapper> <Menu> Menu: [Fichier / Export-Manager] </Menu> </GeoMapper>
<GeoDesigner> <Menu> Menu: [Fichier / Exportation] </Menu> 
 <Menu> Cmd: [ExportManagerOpen] </Menu> </GeoDesigner>

### S&#233;lection du type (formats d&#39;exportation): 
S&#233;lectionnez le format vers lequel vous d&#233;sirez exporter.

### S&#233;lection du fichier
<GeoMapper>

- <Bold> S&#233;lection du fichier pour l&#39;exportation: </Bold>  Selon le format d&#39;exportation on peut donner un nom de fichier ou un dossier.


- <Bold> S&#233;lection des objets &#224; exporter: </Bold>  


- <Bold> Tous les objets existants dans la base de donn&#233;es: </Bold>  Avant d&#39;exporter il faut sauver les modifications. Apr&#232;s, tous le contenu du ficher est export&#233;. Les objets &#233;ventuellement non charg&#233;s sont aussi export&#233;s (voir 

[Charger zone](../datei/bereichladen)
)


- <Bold> Tous les objets charg&#233;s: </Bold>  Tous les objets charg&#233;s dans le graphique


- <Bold> S&#233;lection dans le graphique: </Bold>  Ici on passe directement sur le graphique, pour 

[s&#233;lectionner les objets](../grafik/selektionvondaten)
 &#224; exporter.






<Warning>
Cette option n&#39;est disponible que si les objets &#224; exporter sont charg&#233;s. Si n&#233;cessaire chargez une zone (voir [Charger zone](../datei/bereichladen)
).
</Warning>


- <Bold> Exporter seulement objets visibles: </Bold>  Cochez cette option si vous d&#233;sirez exporter seulement les donn&#233;es visibles sur le graphique. Sinon aussi les objets dont le type a &#233;t&#233; masqu&#233; ou non visibles &#224; cause d&#39;un filtre, seront export&#233;s.

- <Bold> Domaine num&#233;rotation points: </Bold>  Avec cette option vous pouvez d&#233;finir aussi un domaine de num&#233;rotation des points, lequel est combin&#233; avec la s&#233;lection g&#233;n&#233;rale des objets.







<Warning>
Cette option n&#39;est disponible que pour l&#39;exportation  <Italic> Points ASCII </Italic>  et  <Italic> rmGEO </Italic> .
</Warning>


- <Bold> Transformation dans un syst&#232;me de coordonn&#233;es diff&#233;rent: </Bold>  Si l&#39;exportation doit se faire dans un syst&#232;me de coordonn&#233;es diff&#233;rent de celui d&#233;finit dans les param&#232;tres du fichier, le syst&#232;me de coordonn&#233;es de destination peut &#234;tre d&#233;finit ici. Autrement il n&#39;est pas n&#233;cessaire de choisir un syst&#232;me. 



<Warning>
 <Bold> Exportation dans une G&#233;odatabase: </Bold>  Si dans les param&#232;tres du fichier, un facteur de r&#233;duction est d&#233;finit, ce dernier est calcul&#233; &#224; nouveau dans le nouveau syst&#232;me - &#224; condition que le centre de gravit&#233;, l&#39;altitude moyenne et le rayon terrestre soient connus. Autrement le facteur de r&#233;duction est repris. Lors d&#39;une transformations dans un syst&#232;me local, le facteur de r&#233;duction est &#233;videmment d&#233;finit &#224; 1.
</Warning>


- <Bold> Syst&#232;me locale: </Bold>  Les donn&#233;es sont corrig&#233;es avec une transformation Helmert-2D. Comme param&#232;tre de translation est consid&#233;r&#233; le centre de gravit&#233; choisi dans les param&#232;tres du ficher qui sert comme point d&#39;origine. La rotation est z&#233;ro et l&#39;&#233;chelle correspond au facteur de r&#233;duction.






</GeoMapper>
