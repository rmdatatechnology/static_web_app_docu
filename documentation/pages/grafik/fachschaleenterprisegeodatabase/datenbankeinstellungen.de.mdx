## Einstellungen im Viewing-Modus

Die Einstellungen k&#246;nnen f&#252;r ein oder mehrere Geodatenbanken einer Enterprise Geodatabase im Viewingmodus ge&#228;ndert werden.
Dazu wird in den Verbindungseinstellungen ein oder mehrere Datenbanken ausgew&#228;hlt und der Befehl  <Menu>Datenbankeinstellungen bearbeiten</Menu>  aufgerufen.<Image>\img\grafik\fachschaleenterprisegeodatabase\dbeinstellungenbearbeiten.png$dbeinstellungenbearbeiten</Image>

### Konfigurationen
Die Konfigurationen der ausgew&#228;hlten Datenbanken k&#246;nnen aktualisiert, gel&#246;scht oder neue Konfigurationen hinzugef&#252;gt werden.

### Transformation
Das verwendete Koordinatensystem oder eine Helmert-2D-Transformation kann angegeben werden.

### Einstellungen
In den Einstellungen werden die sichtbaren Featureklassen definiert, welche im Darstellungsmanager angezeigt werden.

### Erweiterte Einstellungen
Einstellen von Darstellung, Ansicht und Ma&#223;stab

## Einstellungen im Editiermodus
Die Einstellungen einer Datenbank werden im Editiermodus &#252;ber den Befehl  <Menu>Datenbankeinstellungen bearbeiten</Menu>  aufgerufen.

### Enterprise Geodatenbank

- Verdrehung von Punkten, Texten und Linienzugsymbolen in SDO speichern: Mit dieser Einstellung wird beim Speichern von Objekten zus&#228;tzlich die Verdrehung in den Geometriedaten der Datenbank (SDO-Geometry) gespeichert.

- Angabe eines Bauauftrags notwendig: Beim Bearbeiten von Projektgebieten muss auch ein aktiver Bauauftrag angegeben werden. Dieser wird in der &#196;nderungstabelle mitgespeichert.

- &#220;bersichtsdarstellung laden

- Bearbeitungsma&#223;stab fixieren: Der Bearbeitungsma&#223;stab kann vom Anwender nicht ge&#228;ndert werden. Damit wird sichergestellt, dass Bearbeitungen nicht in einem falschen Ma&#223;stab erfolgen.





- Beim Berechtigen von Projektgebieten nur Gruppen anzeigen: Bei aktivierter Option werden nur die Gruppen aus dem Active Directory zur Auswahl angeboten, jedoch keine einzelnen User.

- &#220;bernommene Projektgebiete nicht l&#246;schen: &#220;bernommene Projektgebiete werden nicht aus der Datenbank gel&#246;scht, sondern bleiben als Objekte vom Typ &quot;Projektgebiet &#252;bernommen&quot; erhalten. So k&#246;nnen nachtr&#228;glich die Einstellungen f&#252;r das Projektgebiet eingesehen werden.

- Beim Erstellen von Projektgebieten Projektgebiet als private GeoDB exportieren: Jedes mal, wenn ein neues Projektgebiet angelegt wird, wird in diesem Bereich der Live-Stand aus der Enterprise-GeoDB in eine private GeoDB exportiert.

- Konfiguration: Projektgebiete k&#246;nnen weiters um benutzerspezifische Attribute erweitert werden. Dazu muss die Konfiguration &quot;LockRegions&quot; (Diese wird defaultm&#228;&#223;ig in den Ordner &quot;%programdata%/rmDATA/GeoDesktop/Configurations&quot; installiert) angepasst und in das Projekt geladen werden. Beim Hinzuf&#252;gen von Attributen m&#252;ssen diese in den Objekttypen &quot;Projektgebiet&quot; und &quot;Projektgebiet &#252;bernommen&quot; konfiguriert werden. Bestehende Attribute d&#252;rfen nicht ver&#228;ndert werden!<Image>\img\grafik\fachschaleenterprisegeodatabase\einstellungengeodatabase.png$einstellungengeodatabase</Image>
