## Anzeigen und Editieren von Daten aus einer rmDATA Enterprise Geodatabase

In einer rmDATA Enterprise Geodatabase werden gro&#223;e Datenmengen, bei Bedarf im Multi-User-Betrieb, verwaltet und bearbeitet.
Die Daten einer rmDATA Enterprise Geodatabase werden in einer zentralen ORACLE- oder SQL-Server-Datenbank gespeichert.

## Fachschale aktivieren
Das Aktivieren und Deaktivieren von Fachschalen erfolgt in den [Dokumenteigenschaften](../einstellungen/dokumenteneinstellungen)
.

## Verbindung zur Datenquelle herstellen
Die Verbindung zur Enterprise Geodatabase wird &#252;ber den [Datenquellenmanager](../datenquellenmanager/allgemein)
 hergestellt.<Image>\img\grafik\fachschaleenterprisegeodatabase\VerbindungEnterpriseGeoDB.png$VerbindungEnterpriseGeoDB</Image>

Es sind folgende Eingaben erforderlich:

- Verbindungsname

- Hostname

- Port

- Servicename

- Benutzername

- Passwort




## Auswahl der Geodatenbanken
Eine Datenbankverbindung kann ein oder mehrere Geodatenbanken beinhalten. Die verf&#252;gbaren Datenbanken werden im Register  <Menu>GeoDatabase</Menu>  angezeigt.<Image>\img\grafik\fachschaleenterprisegeodatabase\geodbauswahl.png$geodbauswahl</Image>

Der Status (Im Geodokument eingebunden, mit Berechtigungen gesch&#252;tzt) jeder Geodatebank ist links vom Namen mit einem Symbol ersichtlich.
Auf der rechten Seite k&#246;nnen

- Skripts auf die gew&#228;hlten Datenbanken ausgef&#252;hrt werden

- die Einstellungen jeder Datenbank (mit Administrator-Rechten) bearbeitet werden

- die Datenbank in das aktuelle Geodokument eingebunden oder ausgeh&#228;ngt werden

- die Rechtverwaltung einer Datenbank aktiviert werden.




Administratoren k&#246;nnen weiters neue Geodatenbanken anlegen sowie bestehende Geodatenbanken l&#246;schen.
Die Rolle des Administrators wird direkt am Oracle-Schema bzw. SQL-Server-Datenbank vergeben.
