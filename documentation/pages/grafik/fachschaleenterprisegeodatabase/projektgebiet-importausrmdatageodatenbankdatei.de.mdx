## Differenzdaten aus einer rmDATA Geodatenbank Datei in ein Projektgebiet importieren

<Success>
Der Import ist nur in der Fachschale &quot;Enterprise Geodatabase&quot; aktiv.
</Success>


&#220;ber diesen Import k&#246;nnen Differenzdaten aus einer rmDATA Geodatenbank Datei importiert werden. Hierzu werden Objekte, die in der Geodatenbank Datei in der Version &quot;neu&quot; vorgefunden werden, je nach Art der &#196;nderung (hinzugef&#252;gt, ge&#228;ndert, gel&#246;scht, ...) in den Datenbestand &#252;bernommen. &#220;ber die IDs der importierten Objekte werden die potenziellen Vorg&#228;nger in der Version &quot;alt&quot; im Datenbestand gesucht. Wurde ein Objekt, dass in der Import-Datei ge&#228;ndert wurde, im Projektgebiet auch ge&#228;ndert, so geht die &#196;nderung im Projektgebiet verloren und jene aus der Import-Datei wird &#252;bernommen.

<Warning>
 Befinden sich in der Import-Datei Linienzugsymbole, werden diese beim Import ignoriert!
</Warning>
