## Erstellen einer eigenen Darstellung f&#252;r Skizze, Riss oder Grenzniederschrift - ohne die urspr&#252;nglichen Daten zu ver&#228;ndern.

<GeoDesigner> <Menu> Menu: [ Daten / Verzerrte Darstellung anlegen] </Menu> </GeoDesigner><GeoMapper> <Menu>Datei / Verzerrte Darstellung anlegen</Menu> </GeoMapper>


1.  Rufen Sie den Befehl auf

2.  Geben Sie den Namen der neuen verzerrten Darstellung ein

3.  W&#228;hlen Sie die Darstellung aus der Konfiguration f&#252;r die passende Visualisierung der Objekte. 


<Success>
Die Darstellung k&#246;nnen Sie sp&#228;ter nicht mehr ver&#228;ndern. Ansichten und Ma&#223;st&#228;be lassen sich aber in der Bearbeitung wie gewohnt w&#228;hlen.

</Success>
4. Klicken Sie auf <Border>Ok</Border> 



<Success>
 <Bold> St&#252;tzpunkte einf&#252;gen </Bold> 
In der verzerrten Darstellung k&#246;nnen Sie in einem Linienzug keine zus&#228;tzlichen St&#252;tzpunkte einf&#252;gen. Daher ist die Einstellung 
<Image>\img\grafik\verzerrtedarstellung\Stützpunkte_aus.png$Stützpunkte_aus</Image> in der Statusleiste ausgegraut.
</Success>

<Success>
 <Bold> Bema&#223;ungen </Bold> 
Bema&#223;ungen k&#246;nnen nur auf Basis bestehender Punkte bzw. St&#252;tzpunkte eingef&#252;gt werden. So ist gew&#228;hrleistet, dass Sie immer die echten Bema&#223;ungswerte sehen - unabh&#228;ngig davon, wie die Daten verzerrt wurden.
</Success>

<Success>
 <Bold> Punkt&#228;nderungsdialog </Bold> 
Beim Verschieben von Punkten wird kein Punkt&#228;nderungsdialog angezeigt, damit Sie rasch eine sprechende Grafik erzeugen k&#246;nnen.
</Success>

<Success>
 <Bold> Erg&#228;nzungen </Bold> 
Werden im Lageplan neue Objekte eingef&#252;gt, dann sind diese ebenso in der verzerrten Darstellung (abh&#228;ngig von der Konfiguration) sichtbar. 
</Success>

<Success>
 <Bold> &#196;nderungen im Lageplan </Bold> 
Wenn im Lageplan ein Punkt ge&#228;ndert wird, dann wird er in der verzerrten Darstellung auf diese Koordinaten zur&#252;ckgesetzt. Auf diese Weise sind &#196;nderungen z.B. von Geb&#228;uden auch in der verzerrten Darstellung sichtbar und m&#252;ssen dort kein zweites Mal konstruiert werden.
</Success>
